  • HTML
    • Basic HTML Structure

      Open Visual Studio code. Create a html file in the folder you open. Then, type “!” and press enter button on your keyboard. A basic structure will be generate in there. Here is the code you will see:

      <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <!-- we will write code here. -->
</body>
</html>

      Document Structure:

      • <!DOCTYPE html>:
        • Declares the document as an HTML5 document. This tells the browser to render the page using the HTML5 standards.
      • <html lang="en">:
        • This is the root element of the HTML document. The lang="en" attribute specifies the language of the document (in this case, English).
      • <head>:
        • Contains meta-information about the document, such as the title, character encoding, and responsive behavior.
      • Inside the <head>:
        • <meta charset="UTF-8">:
          • This tag specifies the character encoding for the document as UTF-8, which is the most widely used encoding. It ensures that the webpage can display a wide range of characters from different languages correctly.
        • <meta name="viewport" content="width=device-width, initial-scale=1.0">:
          • This tag ensures the webpage is responsive on mobile devices. It controls the page's dimensions and scaling, setting the width to the device's width and ensuring a 1:1 scale (no zoom by default). Without this, mobile browsers may display your webpage incorrectly or zoomed out.
        • <title>Document</title>:
          • Defines the title of the document that will appear in the browser tab or title bar. It's also used by search engines to display the page's title in search results.
      • <body>:
        • This contains all the visible content of the webpage, such as text, images, links, buttons, and other elements. Everything you want users to see or interact with will be placed inside this element.

    • Anchor <a> & image <img> tag

      1. Anchor (<a>) Tag:

      • The anchor tag defines a hyperlink, which is used to link from one page to another.
      • The href attribute specifies the URL of the page the link goes to.

      Example:

      <a href="<https://www.example.com>">Visit Example Website</a>

      This creates a clickable link that takes the user to https://www.example.com.

      2. Image (<img>) Tag:

      • The image tag embeds an image in the page.
      • The src attribute specifies the image source (URL or path to the image file), and the alt attribute provides alternative text if the image cannot be displayed.

      Example:

      <img src="<https://via.placeholder.com/150>" alt="Placeholder Image">

      This embeds an image from the provided URL, and if the image fails to load, the alt text "Placeholder Image" will be displayed.

      3. Combining Anchor and Image:

      You can wrap an image in an anchor tag to make the image a clickable link.

      Example:

      <a href="<https://www.example.com>">
    <img src="<https://via.placeholder.com/150>" alt="Clickable Placeholder Image">
</a>

      This creates an image that, when clicked, takes the user to https://www.example.com.

    • Lists <li>

      HTML Lists

      Lists are a great way to group related items together in a structured manner. In HTML, there are three types of lists: ordered lists, unordered lists, and definition lists.

      • 1. Unordered List (<ul>)

        An unordered list displays items as bullet points. It is used when the order of items is not important.

        Syntax:

        <ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</ul>

        Example:

        <ul>
    <li>Apples</li>
    <li>Bananas</li>
    <li>Cherries</li>
</ul>

        Output:

        • Apples
        • Bananas
        • Cherries
      • 2. Ordered List (<ol>)

        An ordered list displays items in a specific sequence, typically with numbers. It is used when the order of items matters.

        Syntax:

        <ol>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ol>

        Example:

        <ol>
    <li>Step 1: Open the browser.</li>
    <li>Step 2: Navigate to the website.</li>
    <li>Step 3: Start exploring.</li>
</ol>

        Output:

        1. Step 1: Open the browser.
        1. Step 2: Navigate to the website.
        1. Step 3: Start exploring.

        Types of Ordered Lists:

        You can change the number format using the type attribute:

        • Numbers (default): <ol type="1">
        • Uppercase Letters: <ol type="A">
        • Lowercase Letters: <ol type="a">
        • Uppercase Roman Numerals: <ol type="I">
        • Lowercase Roman Numerals: <ol type="i">

        Example:

        <ol type="A">
    <li>Introduction</li>
    <li>Content</li>
    <li>Conclusion</li>
</ol>

        Output:

        A. Introduction

        B. Content

        C. Conclusion

      • 3. Definition List (<dl>)

        A definition list is used for terms and descriptions. It contains <dt> (definition term) and <dd> (definition description) elements.

        Syntax:

        <dl>
    <dt>HTML</dt>
    <dd>The standard language for creating web pages.</dd>

    <dt>CSS</dt>
    <dd>A style sheet language used for describing the look of a webpage.</dd>
</dl>

        Example:

        <dl>
    <dt>API</dt>
    <dd>Application Programming Interface</dd>

    <dt>HTTP</dt>
    <dd>Hypertext Transfer Protocol</dd>
</dl>

        Output:

        • API: Application Programming Interface
        • HTTP: Hypertext Transfer Protocol
      • Customizing List Style

        You can modify the appearance of lists using CSS.

        Changing Bullet Style in Unordered List:

        ul {
    list-style-type: square; /* options: circle, disc (default), square, none */
}

        Example:

        <ul style="list-style-type: square;">
    <li>Item 1</li>
    <li>Item 2</li>
</ul>

        Output:

        ■ Item 1

        ■ Item 2

    • Input <input>

      The <input> element in HTML is used to create interactive controls in web forms, allowing users to input data. It is a versatile element that supports various input types, such as text, passwords, email, numbers, etc.

      1. Basic Syntax of Input

      <input type="text" name="inputName">
      • type: Specifies the type of input (e.g., text, email, password).
      • name: Assigns a name to the input field (used in form submission).

      2. Common Input Types

      Below are the most commonly used input types:

      • 2.1 Text Input (type="text")

        Creates a single-line text input field for short, single-line data like names or titles.

        <label for="username">Username:</label>
<input type="text" id="username" name="username">
      • 2.2 Password Input (type="password")

        Hides the characters entered in the field (commonly used for passwords).

        <label for="password">Password:</label>
<input type="password" id="password" name="password">
      • 2.3 Email Input (type="email")

        Validates the input to ensure it's a properly formatted email address.

        <label for="email">Email:</label>
<input type="email" id="email" name="email">
      • 2.4 Number Input (type="number")

        Allows users to input numbers. You can also specify a range using min, max, and step.

        <label for="age">Age:</label>
<input type="number" id="age" name="age" min="1" max="100" step="1">
      • 2.5 Checkbox Input (type="checkbox")

        Creates a checkbox that allows the user to select multiple options.

        <label for="subscribe">Subscribe to Newsletter:</label>
<input type="checkbox" id="subscribe" name="subscribe">
      • 2.6 Radio Input (type="radio")

        Creates radio buttons that allow the user to select only one option from a group.

        <label for="gender">Gender:</label>
<input type="radio" id="male" name="gender" value="male"> Male
<input type="radio" id="female" name="gender" value="female"> Female
      • 2.7 Date Input (type="date")

        Provides a date picker for selecting a date.

        <label for="dob">Date of Birth:</label>
<input type="date" id="dob" name="dob">
      • 2.8 Range Input (type="range")

        Creates a slider control for selecting a value within a range.

        <label for="volume">Volume:</label>
<input type="range" id="volume" name="volume" min="0" max="100">
      • 2.9 File Input (type="file")

        Allows users to upload files.

        <label for="resume">Upload Resume:</label>
<input type="file" id="resume" name="resume">
      • 2.10 Submit Button (type="submit")

        Creates a button that submits the form.

        <input type="submit" value="Submit">
      • 2.11 Reset Button (type="reset")

        Resets all form fields to their initial values.

        <input type="reset" value="Reset">

      3. Input Attributes

      Here are some common attributes you can use with <input> elements:

      • 3.1 placeholder

        Provides a hint to the user about what to enter in the input field.

        <input type="text" placeholder="Enter your name">
      • 3.2 required

        Makes an input field mandatory for form submission.

        <input type="email" required>
      • 3.3 disabled

        Disables an input field, making it non-editable and non-submittable.

        <input type="text" disabled>
      • 3.4 readonly

        Makes an input field read-only, so users can view but not change the content.

        <input type="text" value="This is read-only" readonly>
      • 3.5 value

        Sets the initial value of an input field.

        <input type="text" value="Pre-filled Text">
      • 3.6 maxlength

        Limits the number of characters the user can input.

        <input type="text" maxlength="10">
      • 3.7 min and max

        Specifies the minimum and maximum values for number, date, and range inputs.

        <input type="number" min="1" max="10">
<input type="date" min="2022-01-01" max="2024-12-31">
      • 3.8 step

        Defines the stepping interval for numeric or range input.

        <input type="number" step="5">
<input type="range" step="10">
    • Form <>

      HTML forms are essential elements for collecting user input on a webpage. They allow users to enter data and submit it to a server for processing. The <form> element wraps around input fields and buttons to group them together.

      • 1. Basic Structure of a Form

        A form typically consists of input fields (such as text boxes, checkboxes, radio buttons, etc.) and a submit button.

        Syntax:

        <form action="submit-form.php" method="POST">
    <!-- Input fields go here -->
</form>
        • action: Specifies the URL where the form data should be submitted.
        • method: Specifies the HTTP method to use when submitting the form. The common values are:
          • GET: Sends the form data as part of the URL (useful for searches).
          • POST: Sends the form data as part of the HTTP request body (more secure, commonly used).

        Example:

        <form action="/submit" method="POST">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name">

    <input type="submit" value="Submit">
</form>
      • 2. Form Attributes

        Here are some common attributes used with the <form> element:

        2.1 action

        Defines where the form data will be submitted (URL or file).

        2.2 method

        Defines how the form data will be sent (GET or POST).

        2.3 target

        Specifies where to display the response after form submission. The common values are:

        • _self: Loads the response in the same window (default).
        • _blank: Opens the response in a new tab or window.
        • _parent: Loads the response in the parent frame.
        • _top: Loads the response in the full body of the window.

        Example:

        <form action="/submit" method="POST" target="_blank">
    <!-- form elements -->
</form>

        2.4 autocomplete

        Specifies whether the browser should automatically complete form fields based on previous input. Values: on or off.

        Example:

        <form action="/submit" method="POST" autocomplete="on">
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" autocomplete="off">
</form>

        2.5 novalidate

        Disables form validation by the browser.

        Example:

        <form action="/submit" method="POST" novalidate>
    <!-- form elements -->
</form>
      • 3. Form Elements

        Forms consist of various types of input elements. Here are some of the key ones:

        3.1 Input Fields (<input>)

        As described earlier, input fields allow users to enter data. Examples include text boxes, passwords, email fields, etc.

        <label for="username">Username:</label>
<input type="text" id="username" name="username">

        3.2 Labels (<label>)

        The <label> element provides a caption for input fields and improves accessibility by associating a label with an input.

        <label for="password">Password:</label>
<input type="password" id="password" name="password">

        3.3 Textarea (<textarea>)

        The <textarea> element is used for multi-line text input.

        <label for="bio">Bio:</label>
<textarea id="bio" name="bio" rows="4" cols="50">Enter your bio...</textarea>

        3.4 Select Box (<select>)

        The <select> element creates a drop-down menu with options.

        <label for="country">Country:</label>
<select id="country" name="country">
    <option value="usa">USA</option>
    <option value="canada">Canada</option>
    <option value="uk">UK</option>
</select>

        3.5 Radio Buttons (<input type="radio">)

        Radio buttons allow users to select one option from a set.

        <label>Gender:</label>
<input type="radio" id="male" name="gender" value="male">
<label for="male">Male</label>
<input type="radio" id="female" name="gender" value="female">
<label for="female">Female</label>

        3.6 Checkboxes (<input type="checkbox">)

        Checkboxes allow users to select multiple options.

        <label for="subscribe">Subscribe to Newsletter:</label>
<input type="checkbox" id="subscribe" name="subscribe">

        3.7 Buttons

        The submit, reset, and button elements, as mentioned in the button section, are used to handle user actions.

        <input type="submit" value="Submit">
<input type="reset" value="Reset">
      • 4. Form Validation

        HTML forms include built-in validation to ensure that the submitted data meets the specified requirements.

        4.1 Required Fields

        Use the required attribute to make a field mandatory.

        <input type="text" name="username" required>

        4.2 Pattern Matching

        The pattern attribute allows you to specify a regular expression that the input must match.

        <input type="text" name="zipcode" pattern="\\d{5}" title="Enter a valid 5-digit ZIP code">

        4.3 Input Length

        The minlength and maxlength attributes define the minimum and maximum length of input.

        <input type="text" name="username" minlength="3" maxlength="15">

        4.4 Number Input Limits

        The min, max, and step attributes restrict the range of numeric inputs.

        <input type="number" name="age" min="18" max="100" step="1">
      • 5. Form Submission Methods

        Forms can submit data in two ways:

        5.1 GET Method

        • Sends form data appended to the URL as query parameters.
        • Useful for non-sensitive data (like search queries).

        Example:

        <form action="/search" method="GET">
    <input type="text" name="query">
    <input type="submit" value="Search">
</form>

        URL Example: /search?query=example

        5.2 POST Method

        • Sends form data in the request body.
        • Ideal for sensitive data (like login forms) since it is not visible in the URL.

        Example:

        <form action="/login" method="POST">
    <input type="text" name="username">
    <input type="password" name="password">
    <input type="submit" value="Login">
</form>
      • 6. File Upload Forms

        The enctype="multipart/form-data" attribute is required to handle file uploads.

        Example:

        <form action="/upload" method="POST" enctype="multipart/form-data">
    <label for="file">Choose a file:</label>
    <input type="file" id="file" name="file">
    <input type="submit" value="Upload">
</form>

  • CSS
    • 1. Introduction to CSS
      • Definition: CSS (Cascading Style Sheets) is used to control the presentation and layout of web pages.
      • Purpose: CSS separates content (HTML) from design, making it easier to manage the appearance of web pages across different devices and screen sizes.
      • Cascading: CSS applies styles according to a priority order of rules. If multiple rules apply to the same element, the most specific rule takes precedence.
    • 2. CSS Syntax and Structure

      CSS (Cascading Style Sheets) is written in rulesets that define how HTML elements should be displayed. The basic structure of a CSS rule includes selectors, properties, and values.

      • Basic CSS Rule Syntax:
        selector {
    property: value;
    property: value;
}
        • Selector: Specifies the HTML element(s) you want to style.
        • Property: Defines the style aspect you want to change (e.g., color, margin).
        • Value: The setting for the property (e.g., red, 20px).
      • 1. Selectors

        CSS selectors are used to target the HTML elements you want to apply styles to. There are different types of selectors, each designed to target elements in a specific way.

        • 1.1 Element Selector

          The element selector applies styles to all instances of the given HTML element.

          • Example:
            p {
    color: blue;
    font-size: 16px;
}

            This CSS rule targets all <p> (paragraph) elements, turning the text color blue and setting the font size to 16px.

        • 1.2 Class Selector

          The class selector applies styles to all elements with a specific class attribute. Class selectors start with a period (.).

          • Example:
            .card {
    border: 1px solid black;
    padding: 20px;
}

            This rule applies to any HTML element with the class="card", adding a black border and padding.

            HTML:

            <div class="card">This is a card.</div>
        • 1.3 ID Selector

          The ID selector targets a specific element with a unique id attribute. ID selectors are prefixed with a hash (#). IDs must be unique within a page.

          • Example:
            #header {
    background-color: grey;
    padding: 10px;
}

            This rule styles the element with id="header" by adding a grey background and padding.

            HTML:

            <div id="header">This is the header.</div>
        • 1.4 Universal Selector

          The universal selector (*) targets all elements on the page.

          • Example:
            * {
    margin: 0;
    padding: 0;
}

            This rule removes all default margins and padding from every element on the page.

        • 1.5 Group Selector

          The group selector allows you to apply the same style to multiple elements. You can group selectors by separating them with commas.

          • Example:
            h1, h2, p {
    color: black;
    font-family: Arial, sans-serif;
}

            This rule applies the same text color and font family to all <h1>, <h2>, and <p> elements.

        • 1.6 Attribute Selector

          Attribute selectors target elements based on the presence of an attribute or the value of an attribute.

          • Example:
            input[type="text"] {
    border: 2px solid blue;
}

            This rule applies to any <input> element with the attribute type="text", giving it a blue border.

            HTML:

            <input type="text" placeholder="Enter your name">
<input type="password" placeholder="Enter your password">

            Only the text input gets the blue border.

        • 1.7 Descendant Selector

          The descendant selector applies styles to elements nested inside a specific element.

          • Example:
            div p {
    color: green;
}

            This rule applies to all <p> elements inside a <div>, turning their text green.

            HTML:

            <div>
    <p>This paragraph is inside a div.</p>
</div>
<p>This paragraph is not inside a div.</p>

            Only the first paragraph is green.

        • 1.8 Child Selector

          The child selector (>) applies styles to direct child elements of a specific parent element.

          • Example:
            ul > li {
    color: red;
}

            This rule targets only the <li> elements that are direct children of a <ul>, turning their text red.

            HTML:

            <ul>
    <li>First item</li>
    <li>Second item</li>
</ul>
<ol>
    <li>First item in ordered list</li>
</ol>

            Only the <li> elements inside the unordered list (<ul>) will be styled.

        • 1.9 Sibling Selector

          Sibling selectors target elements that share the same parent. There are two types:

          • Adjacent Sibling Selector (+): Targets the next immediate sibling.
          • General Sibling Selector (~): Targets all siblings that follow an element.
          • Adjacent Sibling Example:
            h2 + p {
    color: purple;
}

            This rule applies to the first <p> element that immediately follows any <h2>.

            HTML:

            <h2>Section Title</h2>
<p>This paragraph is purple because it's immediately after an h2.</p>
<p>This paragraph is not affected.</p>
          • General Sibling Example:
            h2 ~ p {
    color: orange;
}

            This rule applies to all <p> elements that follow any <h2> (not just the first one).

            HTML:

            <h2>Section Title</h2>
<p>This paragraph is orange.</p>
<p>This paragraph is also orange.</p>

          Applies styles when an element (typically a form input or a button) receives focus, either by clicking or using the keyboard (e.g., tabbing).

          <input type="text" placeholder="Click or tab to focus">

    • 6. CSS Units

      CSS uses various units to define the size of elements, spacing, fonts, and more. These units can be classified into two categories: absolute and relative units.

      • 1. Absolute Units

        Absolute units represent fixed sizes, meaning they do not change based on the screen size, parent element, or any other external factors. These units are ideal when you want elements to have a precise size regardless of the display or device.

        Common Absolute Units:

        • px (Pixels): The most commonly used unit, representing one dot on the screen. It’s fixed and remains the same across devices.
        • cm (Centimeters): Unit measured in centimeters, useful for printed documents but not commonly used for screen display.
        • mm (Millimeters): Unit measured in millimeters.
        • in (Inches): Unit measured in inches, generally used for print media.
        • pt (Points): Used in typography, 1 point is 1/72 of an inch.
        • pc (Picas): Another unit used in typography; 1 pica equals 12 points.

        Example of Absolute Units:

        div {
    width: 300px;
    height: 200px;
    padding: 10cm;
    margin: 1in;
}
        • The width and height are set in pixels (px).
        • Padding is set in centimeters (cm).
        • Margin is set in inches (in).

        Absolute Unit Use Cases:

        • Useful for scenarios where precise sizes are required (e.g., fixed layout designs, print media).
        • Drawback: Not responsive, meaning elements won’t adjust well to different screen sizes.
      • 2. Relative Units

        Relative units are flexible and scale based on the size of other elements, like the parent element or the viewport (browser window). These units are critical for building responsive designs that adapt to different screen sizes and devices.

        Common Relative Units:

        1. em

        • Relative to the font-size of the element itself or the parent element.
        • 1em equals the current font size of the element. If the font size of a parent element is set to 16px, then 1em equals 16px.
        • Useful for creating elements that scale based on the parent font size.

        Example:

        p {
    font-size: 1.5em;  /* 1.5 times the parent font size */
}

        2. rem

        • Relative to the root element’s font-size (<html> element). By default, most browsers set the root font-size to 16px.
        • 1rem equals the font size of the root element, usually 16px.

        Example:

        h1 {
    font-size: 2rem;  /* 2 times the root font size, typically 32px */
}
        • Difference between em and rem: em scales based on the parent element, while rem is consistent and scales based on the root font size.

        3. % (Percentage)

        • Percentages are relative to the parent element’s size.
        • If you set width: 50%, the element will take up 50% of its parent element's width.

        Example:

        div {
    width: 50%;  /* Takes up 50% of the parent container's width */
}

        4. vw (Viewport Width)

        • 1vw equals 1% of the viewport’s width. The viewport is the visible area of the browser window.
        • As the window resizes, the vw value adjusts accordingly.

        Example:

        div {
    width: 50vw;  /* Takes up 50% of the viewport's width */
}

        5. vh (Viewport Height)

        • 1vh equals 1% of the viewport’s height.
        • Similar to vw, the vh unit adapts to the viewport’s height.

        Example:

        div {
    height: 100vh;  /* Full height of the viewport */
}

        6. vmin and vmax

        • vmin: Uses the smaller of either the viewport’s width or height. For example, if the viewport width is 800px and the height is 600px, 1vmin would be 6px (1% of 600px).
        • vmax: Uses the larger of either the viewport’s width or height. In the same case, 1vmax would be 8px (1% of 800px).

        Example:

        div {
    width: 50vmin;  /* 50% of the smaller dimension of the viewport */
}

        7. ex and ch

        • ex: Relative to the height of the lowercase letter "x" in the font.
        • ch: Relative to the width of the "0" (zero) character in the font. Useful for creating text inputs or monospaced designs.

        Example:

        input {
    width: 10ch;  /* Width equals the size of 10 "0" characters */
}

        Unit TypeCommon UnitsBehaviorUse Cases
        Absolute Unitspx, cm, in, ptFixed and does not change with screen sizeIdeal for print design, static, or pixel-perfect layouts
        Relative Unitsem, rem, %, vw, vhScales relative to another value (parent element or viewport)Best for responsive web design, flexible layouts, dynamic resizing
    • 7. CSS Colors

      In CSS, colors can be applied to elements using different notations. You can use predefined color names or specify colors using hexadecimal, RGB/RGBA, or HSL/HSLA values. Each method provides a different way to define colors, depending on your needs for accuracy, transparency, or flexibility.

      • 1. Named Colors

        CSS provides a set of predefined color names that are easy to use. These names are intuitive and include common colors like red, blue, green, and many more.

        Key Points:

        • Simple and easy to remember.
        • Limited to a predefined set of common colors.

        Example:

        p {
    color: red;
}

        Common Named Colors:

        • black
        • white
        • red
        • green
        • blue
        • yellow
        • cyan
        • magenta
        • purple
        • orange

        There are over 140 named colors available in CSS.

      • 2. Hexadecimal Notation

        Hexadecimal (hex) notation represents colors using a 6-digit or 3-digit hexadecimal number. Each pair of digits represents the intensity of red, green, and blue, ranging from 00 (lowest) to FF (highest).

        Key Points:

        • More precise than named colors.
        • Often used for web design because of its wide support and flexibility.
        • Follows the format: #RRGGBB.

        Example (6-digit):

        h1 {
    color: #ff0000;  /* Red */
}

        In this example:

        • #ff0000: The first two digits (ff) represent red.
        • The second pair (00) represents green.
        • The third pair (00) represents blue.

        3-Digit Hex:

        You can also use a shorthand 3-digit hex notation when the values are the same for both digits in each pair.

        Example:

        h1 {
    color: #f00;  /* Red */
}

        This is equivalent to #ff0000.

      • 3. RGB and RGBA

        The RGB color model defines colors using three values: red, green, and blue. Each value ranges from 0 (none of that color) to 255 (maximum of that color). RGBA adds an additional alpha channel to control the color’s transparency.

        Key Points:

        • Allows more precise control of colors by setting individual values for red, green, and blue.
        • RGBA is useful for adding transparency, making it perfect for overlays and backgrounds.

        RGB Syntax:

        h2 {
    color: rgb(255, 0, 0);  /* Red */
}
        • rgb(255, 0, 0) produces a solid red color by setting the red value to 255, while green and blue are both set to 0.

        RGBA Syntax:

        div {
    background-color: rgba(255, 0, 0, 0.5);  /* Semi-transparent red */
}
        • rgba(255, 0, 0, 0.5) produces a red color with 50% opacity (alpha channel is 0.5, meaning 50% transparent).

        Alpha Channel Values:

        • The alpha value ranges from 0 (completely transparent) to 1 (completely opaque).
      • 4. HSL and HSLA

        The HSL color model represents colors using three values:

        • Hue: The type of color, ranging from 0 to 360 degrees on the color wheel (0 is red, 120 is green, 240 is blue).
        • Saturation: How pure the color is, from 0% (gray) to 100% (full color).
        • Lightness: How light or dark the color is, from 0% (black) to 100% (white).

        HSLA adds an alpha channel for transparency, just like RGBA.

        Key Points:

        • HSL is useful when you want to work with color theory, as you can easily adjust hue, saturation, and lightness.
        • HSLA adds transparency control.

        HSL Syntax:

        p {
    color: hsl(0, 100%, 50%);  /* Red */
}
        • hsl(0, 100%, 50%): This produces a pure red color.
          • Hue: 0 (Red).
          • Saturation: 100% (full intensity).
          • Lightness: 50% (normal brightness).

        HSLA Syntax:

        div {
    background-color: hsla(0, 100%, 50%, 0.5);  /* Semi-transparent red */
}
        • hsla(0, 100%, 50%, 0.5) produces a semi-transparent red, with 50% opacity.

    • 8. CSS Fonts and Text Styling

      CSS offers a variety of properties to control how text appears on a webpage. These properties allow you to define fonts, control the size, weight, style, alignment, and more. Proper text styling is crucial for both readability and design.

      • 1. Font Family

        The font-family property specifies the typeface to be used for the text. You can define a primary font and provide fallback fonts in case the primary font is unavailable. If the font name consists of multiple words, you must wrap it in quotes.

        Syntax:

        font-family: "Times New Roman", Arial, sans-serif;
        • Font Stacks: You can specify multiple fonts, where the browser will use the first available one.
        • Generic Fonts: At the end of your list, it’s a good idea to include a generic family (e.g., serif, sans-serif, monospace) as a fallback.

        Example:

        p {
    font-family: "Arial", Helvetica, sans-serif;
}
      • 2. Font Size

        The font-size property controls the size of the text. You can define font sizes using various units, including px (pixels), em (relative to the parent element), rem (relative to the root element), or %.

        Syntax:

        font-size: 16px;
        • Absolute Units: px for fixed sizing.
        • Relative Units: em or rem for scalable sizing, making the design more responsive.

        Example:

        h1 {
    font-size: 2em;  /* 2 times the size of the parent font size */
}
      • 3. Font Weight

        The font-weight property specifies the thickness of the text. You can use predefined values such as normal or bold, or numeric values ranging from 100 to 900, where 400 is normal and 700 is bold.

        Syntax:

        font-weight: bold;
        • Lighter Text: Use values less than 400.
        • Heavier Text: Use values greater than 400.

        Example:

        h2 {
    font-weight: 700;  /* Bold text */
}
      • 4. Font Style

        The font-style property controls the style of the font. Common values are normal, italic, and oblique.

        • Normal: The default, upright text.
        • Italic: A slanted version of the text.
        • Oblique: Similar to italic but can be artificially slanted.

        Syntax:

        font-style: italic;

        Example:

        blockquote {
    font-style: italic;
}
      • 5. Text Alignment

        The text-align property specifies the horizontal alignment of the text within its container. The common values are left, right, center, and justify.

        • Left: Aligns text to the left (default for left-to-right languages).
        • Right: Aligns text to the right.
        • Center: Centers the text within the element.
        • Justify: Spreads the text so that each line has equal width, with straight edges on both sides.

        Syntax:

        text-align: center;

        Example:

        p {
    text-align: justify;  /* Full-width justified text */
}
      • 6. Text Decoration

        The text-decoration property adds decorative elements to the text, such as underlining, overlining, or striking through the text.

        • underline: Adds a line below the text.
        • line-through: Draws a line through the middle of the text.
        • overline: Adds a line above the text.

        Syntax:

        text-decoration: underline;

        Example:

        a {
    text-decoration: none;  /* Removes the underline from links */
}
      • 7. Line Height

        The line-height property controls the vertical spacing between lines of text within an element. It's often set as a unitless number, percentage, or relative to the font size.

        Syntax:

        line-height: 1.5;  /* 1.5 times the font size */
        • Unitless Value: Scales proportionally with the font size.
        • Percentage or Unit Value: Provides more precise control.

        Example:

        p {
    line-height: 1.6;  /* Adds more spacing between lines */
}
      • 8. Text Transformation

        The text-transform property changes the case of text. The common values are uppercase, lowercase, and capitalize.

        • Uppercase: Converts all text to uppercase.
        • Lowercase: Converts all text to lowercase.
        • Capitalize: Capitalizes the first letter of each word.

        Syntax:

        text-transform: uppercase;

        Example:

        h3 {
    text-transform: capitalize;  /* Capitalizes the first letter of each word */
}
      • 9. Letter Spacing

        The letter-spacing property adjusts the space between characters in the text. Positive values increase the space, while negative values decrease it.

        Syntax:

        letter-spacing: 2px;
        • Useful for improving readability, especially with larger text or when using certain fonts.

        Example:

        h1 {
    letter-spacing: 3px;  /* Adds 3px space between each letter */
}
    • 9. CSS Backgrounds

      The background properties in CSS allow you to control the background color, images, and positioning of those images for any HTML element. They provide significant flexibility in creating visually appealing designs.

      • 1. Background Color

        The background-color property sets the background color of an element. You can specify the color using named colors, hexadecimal values, RGB, RGBA, HSL, or HSLA.

        Syntax:

        background-color: color;

        Example:

        div {
    background-color: lightblue;
}

        This applies a light blue background to the div element.

        Key Points:

        • It covers the entire element's background, including padding (but not the margin).
        • You can use transparency with rgba or hsla.
      • 2. Background Image

        The background-image property allows you to set an image as the background of an element. You specify the image using the url() function.

        Syntax:

        background-image: url('path-to-image');

        Example:

        div {
    background-image: url('background.jpg');
}

        This will set background.jpg as the background image of the div.

        Key Points:

        • The image will be placed behind the element's content.
        • The image can repeat by default, depending on the size of the element and the image.
      • 3. Background Repeat

        The background-repeat property controls whether and how the background image repeats.

        Values:

        • repeat: The image repeats both horizontally and vertically (default behavior).
        • no-repeat: The image will not repeat.
        • repeat-x: The image repeats only horizontally.
        • repeat-y: The image repeats only vertically.
        • space: The image will repeat as many times as possible without clipping, with gaps between the images.
        • round: The image will repeat, but the last image will be resized to avoid clipping.

        Syntax:

        background-repeat: no-repeat;

        Example:

        div {
    background-image: url('background.jpg');
    background-repeat: no-repeat;
}

        This prevents the background image from repeating.

      • 4. Background Size

        The background-size property defines the size of the background image. It controls whether the image stretches, resizes, or maintains its original dimensions.

        Values:

        • auto: Keeps the original size of the image (default).
        • cover: Scales the image to cover the entire element, maintaining aspect ratio.
        • contain: Scales the image to be fully contained within the element, maintaining aspect ratio.
        • Specific values: You can also specify sizes using units like px, %, or em.

        Syntax:

        background-size: cover;

        Example:

        div {
    background-image: url('background.jpg');
    background-size: cover;
}

        This scales the background image to cover the entire div without distorting it.

      • 5. Background Position

        The background-position property sets the starting position of the background image within the element. You can position the background using keywords or specific values.

        Values:

        • Keywords: top, bottom, left, right, center.
        • Coordinates: You can also use values like 10px 20px (first value for horizontal, second for vertical positioning).
        • Percentage: Positioning with percentages (e.g., 50% 50%) places the image at a relative position within the element.

        Syntax:

        background-position: center;

        Example:

        div {
    background-image: url('background.jpg');
    background-position: top right;
}

        This places the background image at the top-right corner of the div.

      • 6. Background Attachment

        The background-attachment property controls whether the background image scrolls with the rest of the content or stays fixed in place.

        Values:

        • scroll: The background image moves as the user scrolls the page (default behavior).
        • fixed: The background image stays fixed in place, even when the user scrolls.
        • local: The background image scrolls with the element's content (useful for scrollable elements).

        Syntax:

        background-attachment: fixed;

        Example:

        div {
    background-image: url('background.jpg');
    background-attachment: fixed;
}

        This makes the background image remain fixed when the user scrolls.

      • Combined Example

        You can combine multiple background properties in a single rule. Here's an example that includes color, image, position, size, and repeat properties:

        div {
    background-color: #f0f0f0;
    background-image: url('background.jpg');
    background-repeat: no-repeat;
    background-position: center;
    background-size: cover;
    background-attachment: fixed;
}

        This example applies:

        • A light grey background color.
        • A background image that does not repeat.
        • The image is centered and covers the element completely.
        • The image remains fixed when the user scrolls.
    • 10. CSS Borders

      CSS borders are used to define the outline or frame around an HTML element. You can control the appearance of borders by setting the style, width, color, and radius to create visually appealing boxes.

      • 1. Border Style

        The border-style property specifies the pattern or style of the border. This is required to make the border visible, even if you’ve set width or color.

        Common Border Styles:

        • solid: A solid, continuous line.
        • dashed: A border made up of dashes.
        • dotted: A border made up of dots.
        • double: Two solid lines with space between them.
        • groove: Creates a carved or indented effect.
        • ridge: Similar to groove, but appears to come out of the page.
        • none: No border is applied.
        • hidden: Like none, but used for table layouts.

        Syntax:

        border-style: solid;

        Example:

        div {
    border-style: dashed;
}

        This applies a dashed border to the div.

      • 2. Border Width

        The border-width property defines the thickness of the border. You can set the width for all sides at once, or specify different widths for each side.

        Syntax:

        border-width: 5px;

        Example:

        p {
    border-width: 2px 4px;  /* 2px on top and bottom, 4px on left and right */
}

        Key Points:

        • You can use absolute units like px, em, or rem.
        • You can apply different widths for the top, right, bottom, and left using four values (e.g., border-width: 2px 4px 6px 8px;).
      • 3. Border Color

        The border-color property specifies the color of the border. You can use named colors, hexadecimal values, RGB, RGBA, HSL, or HSLA for specifying the color.

        Syntax:

        border-color: red;

        Example:

        h1 {
    border-color: #ff0000;
}

        Key Points:

        • You can set the same color for all four sides or specify different colors for each side.
        • If no border-color is specified, the color of the text will be used by default.

        Example (different colors for each side):

        div {
    border-color: red green blue yellow;  /* Top red, right green, bottom blue, left yellow */
}
      • 4. Border Radius

        The border-radius property is used to round the corners of an element. It can be applied equally to all corners or specified separately for each corner.

        Syntax:

        border-radius: 10px;

        Example:

        button {
    border-radius: 50px;  /* Makes the button round */
}

        Key Points:

        • You can control each corner individually by specifying four values (top-left, top-right, bottom-right, bottom-left).
        • You can use percentages for circular shapes (e.g., border-radius: 50% creates a perfect circle if the element is a square).

        Example (different radii for each corner):

        div {
    border-radius: 10px 20px 30px 40px;  /* Top-left: 10px, Top-right: 20px, Bottom-right: 30px, Bottom-left: 40px */
}
      • Combined Example

        You can combine these border properties to define a complete border style in a single declaration, or you can define them individually.

        Example:

        div {
    border-style: solid;
    border-width: 5px;
    border-color: blue;
    border-radius: 10px;
}

        This will create a solid blue border that is 5px thick with 10px rounded corners.

      • Shorthand Property

        You can use the shorthand border property to set the border style, width, and color all in one line.

        Syntax:

        border: 5px solid blue;

        This shorthand sets the border width to 5px, the style to solid, and the color to blue.

    11. CSS Margins and Padding

    • Margin: Defines the space outside the border of an element.
      • margin-top, margin-right, margin-bottom, margin-left, or shorthand margin.
    • Padding: Defines the space between the content and the border of an element.
      • padding-top, padding-right, padding-bottom, padding-left, or shorthand padding.

    12. CSS Display and Visibility

    • Display Property: Defines how an element is displayed.
      • block, inline, inline-block, none, flex, grid.
    • Visibility Property: Controls whether an element is visible or hidden, while keeping its space in the layout (visible, hidden).

    13. CSS Positioning

    • Static Positioning: The default positioning; elements are placed in the normal document flow.
    • Relative Positioning: Positioned relative to its normal position.
    • Absolute Positioning: Positioned relative to the nearest positioned ancestor.
    • Fixed Positioning: Positioned relative to the browser window and does not move when scrolling.
    • Sticky Positioning: A hybrid between relative and fixed positioning, where the element becomes fixed based on the scroll position.

    14. CSS Float and Clear

    • Float: Allows elements to float to the left or right, causing text or other elements to wrap around it.
      • float: left, float: right.
    • Clear: Prevents elements from wrapping around floated elements.
      • clear: left, clear: right, clear: both.

    15. CSS Flexbox

    • Flexbox Layout Model: Provides a more efficient way to lay out, align, and distribute space among elements in a container.
      • display: flex creates a flex container.
      • Main Properties: flex-direction, flex-wrap, justify-content, align-items, align-content, flex-grow, flex-shrink.
      • Flex Items: Controlled individually by order, align-self, flex-basis.

    16. CSS Grid

    • Grid Layout Model: A two-dimensional system for creating grid-based layouts.
      • display: grid creates a grid container.
      • Grid Properties: grid-template-columns, grid-template-rows, grid-column, grid-row, grid-gap, align-items, justify-items.

    17. CSS Transitions and Animations

    • Transitions: Allows smooth change of properties over time.
      • transition-property, transition-duration, transition-timing-function, transition-delay.
    • Animations: Provides more control over how CSS properties change over time.
      • Keyframes: Define the start and end states, as well as intermediate steps.
      • animation-name, animation-duration, animation-timing-function, animation-delay, animation-iteration-count.

    18. CSS Media Queries

    • Media Queries: Used for creating responsive designs by applying styles based on device characteristics (e.g., screen width, orientation).
      • Common breakpoints: max-width, min-width.
      • Media Features: width, height, orientation, aspect-ratio.

    19. CSS Variables (Custom Properties)

    • CSS Variables: Allows the reuse of property values throughout the stylesheet.
      • Defined with -variable-name, and accessed using var(--variable-name).

    20. CSS Z-Index

    • Z-Index: Controls the stacking order of elements (which element appears on top of another) when elements overlap.

    21. CSS Overflow

    • Overflow Property: Controls what happens when the content of an element overflows its box.
      • overflow: visible, overflow: hidden, overflow: scroll, overflow: auto.

    22. CSS Shorthand Properties

    • Shorthand Properties: Combine multiple CSS properties into a single declaration.
      • margin, padding, border, background, font, transition, animation.

    23. CSS Specificity

    • CSS Specificity: Determines which CSS rule is applied when multiple rules target

    the same element.

    • Calculated based on the types of selectors used (ID selectors > Class selectors > Element selectors).
    • Important Rule: !important can override specificity but should be used sparingly.

    24. CSS Inheritance

    • Inheritance: Some properties are inherited from parent elements (e.g., color, font-family).
      • Non-inherited properties (e.g., margin, padding) must be explicitly defined.

    25. CSS Resets and Normalization

    • CSS Reset: Resets all browser default styles to provide a clean slate for your own styles.
    • CSS Normalize: Aims to make built-in browser styling consistent across different browsers without removing all default styles.
