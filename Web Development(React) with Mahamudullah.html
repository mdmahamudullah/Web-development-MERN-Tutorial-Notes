    • Basic HTML Structure

      Open Visual Studio code. Create a html file in the folder you open. Then, type “!” and press enter button on your keyboard. A basic structure will be generate in there. Here is the code you will see:

      <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <!-- we will write code here. -->
</body>
</html>

      Document Structure:

      • <!DOCTYPE html>:
        • Declares the document as an HTML5 document. This tells the browser to render the page using the HTML5 standards.
      • <html lang="en">:
        • This is the root element of the HTML document. The lang="en" attribute specifies the language of the document (in this case, English).
      • <head>:
        • Contains meta-information about the document, such as the title, character encoding, and responsive behavior.
      • Inside the <head>:
        • <meta charset="UTF-8">:
          • This tag specifies the character encoding for the document as UTF-8, which is the most widely used encoding. It ensures that the webpage can display a wide range of characters from different languages correctly.
        • <meta name="viewport" content="width=device-width, initial-scale=1.0">:
          • This tag ensures the webpage is responsive on mobile devices. It controls the page's dimensions and scaling, setting the width to the device's width and ensuring a 1:1 scale (no zoom by default). Without this, mobile browsers may display your webpage incorrectly or zoomed out.
        • <title>Document</title>:
          • Defines the title of the document that will appear in the browser tab or title bar. It's also used by search engines to display the page's title in search results.
      • <body>:
        • This contains all the visible content of the webpage, such as text, images, links, buttons, and other elements. Everything you want users to see or interact with will be placed inside this element.

      • The anchor tag defines a hyperlink, which is used to link from one page to another.
      • The href attribute specifies the URL of the page the link goes to.

      2. Image (<img>) Tag:

      • The image tag embeds an image in the page.
      • The src attribute specifies the image source (URL or path to the image file), and the alt attribute provides alternative text if the image cannot be displayed.

      Example:

      <img src="<https://via.placeholder.com/150>" alt="Placeholder Image">

      This embeds an image from the provided URL, and if the image fails to load, the alt text "Placeholder Image" will be displayed.

      HTML Lists

      Lists are a great way to group related items together in a structured manner. In HTML, there are three types of lists: ordered lists, unordered lists, and definition lists.

      • 1. Unordered List (<ul>)

        An unordered list displays items as bullet points. It is used when the order of items is not important.

        Syntax:

        <ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</ul>

        Example:

        <ul>
    <li>Apples</li>
    <li>Bananas</li>
    <li>Cherries</li>
</ul>

        Output:

        • Apples
        • Bananas
        • Cherries
      • 2. Ordered List (<ol>)

        An ordered list displays items in a specific sequence, typically with numbers. It is used when the order of items matters.

        Syntax:

        <ol>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ol>

        Example:

        <ol>
    <li>Step 1: Open the browser.</li>
    <li>Step 2: Navigate to the website.</li>
    <li>Step 3: Start exploring.</li>
</ol>

        Output:

        1. Step 1: Open the browser.
        1. Step 2: Navigate to the website.
        1. Step 3: Start exploring.

        Types of Ordered Lists:

        You can change the number format using the type attribute:

        • Numbers (default): <ol type="1">
        • Uppercase Letters: <ol type="A">
        • Lowercase Letters: <ol type="a">
        • Uppercase Roman Numerals: <ol type="I">
        • Lowercase Roman Numerals: <ol type="i">

        Example:

        <ol type="A">
    <li>Introduction</li>
    <li>Content</li>
    <li>Conclusion</li>
</ol>

        Output:

        A. Introduction

        B. Content

        C. Conclusion

      • 3. Definition List (<dl>)

        A definition list is used for terms and descriptions. It contains <dt> (definition term) and <dd> (definition description) elements.

        Syntax:

        <dl>
    <dt>HTML</dt>
    <dd>The standard language for creating web pages.</dd>

    <dt>CSS</dt>
    <dd>A style sheet language used for describing the look of a webpage.</dd>
</dl>

        Example:

        <dl>
    <dt>API</dt>
    <dd>Application Programming Interface</dd>

    <dt>HTTP</dt>
    <dd>Hypertext Transfer Protocol</dd>
</dl>

        Output:

        • API: Application Programming Interface
        • HTTP: Hypertext Transfer Protocol
      • Customizing List Style

        You can modify the appearance of lists using CSS.

        Changing Bullet Style in Unordered List:

    • Input <input>

      The <input> element in HTML is used to create interactive controls in web forms, allowing users to input data. It is a versatile element that supports various input types, such as text, passwords, email, numbers, etc.

      1. Basic Syntax of Input

      <input type="text" name="inputName">
      • type: Specifies the type of input (e.g., text, email, password).
      • name: Assigns a name to the input field (used in form submission).

      2. Common Input Types

      Below are the most commonly used input types:

      • 2.1 Text Input (type="text")

        Creates a single-line text input field for short, single-line data like names or titles.

        <label for="username">Username:</label>
<input type="text" id="username" name="username">
      • 2.2 Password Input (type="password")

        Hides the characters entered in the field (commonly used for passwords).

        <label for="password">Password:</label>
<input type="password" id="password" name="password">
      • 2.3 Email Input (type="email")

        Validates the input to ensure it's a properly formatted email address.

        <label for="email">Email:</label>
<input type="email" id="email" name="email">
      • 2.4 Number Input (type="number")

        Allows users to input numbers. You can also specify a range using min, max, and step.

        <label for="age">Age:</label>
<input type="number" id="age" name="age" min="1" max="100" step="1">
      • 2.5 Checkbox Input (type="checkbox")

        Creates a checkbox that allows the user to select multiple options.

        <label for="subscribe">Subscribe to Newsletter:</label>
<input type="checkbox" id="subscribe" name="subscribe">
      • 2.6 Radio Input (type="radio")

        Creates radio buttons that allow the user to select only one option from a group.

        <label for="gender">Gender:</label>
<input type="radio" id="male" name="gender" value="male"> Male
<input type="radio" id="female" name="gender" value="female"> Female
      • 2.7 Date Input (type="date")

        Provides a date picker for selecting a date.

        <label for="dob">Date of Birth:</label>
<input type="date" id="dob" name="dob">
      • 2.8 Range Input (type="range")

        Creates a slider control for selecting a value within a range.

        <label for="volume">Volume:</label>
<input type="range" id="volume" name="volume" min="0" max="100">
      • 2.9 File Input (type="file")

        Allows users to upload files.

        <label for="resume">Upload Resume:</label>
<input type="file" id="resume" name="resume">
      • 2.10 Submit Button (type="submit")

        Creates a button that submits the form.

        <input type="submit" value="Submit">
      • 2.11 Reset Button (type="reset")

        Resets all form fields to their initial values.

        <input type="reset" value="Reset">

      3. Input Attributes

      Here are some common attributes you can use with <input> elements:

      • 3.1 placeholder

        Provides a hint to the user about what to enter in the input field.

        <input type="text" placeholder="Enter your name">
      • 3.2 required

        Makes an input field mandatory for form submission.

        <input type="email" required>
      • 3.3 disabled

        Disables an input field, making it non-editable and non-submittable.

        <input type="text" disabled>
      • 3.4 readonly

        Makes an input field read-only, so users can view but not change the content.

        <input type="text" value="This is read-only" readonly>
      • 3.5 value

        Sets the initial value of an input field.

        <input type="text" value="Pre-filled Text">
      • 3.6 maxlength

        Limits the number of characters the user can input.

        <input type="text" maxlength="10">
      • 3.7 min and max

        Specifies the minimum and maximum values for number, date, and range inputs.

        <input type="number" min="1" max="10">
<input type="date" min="2022-01-01" max="2024-12-31">
      • 3.8 step

        Defines the stepping interval for numeric or range input.

        <input type="number" step="5">
<input type="range" step="10">
    • Form <>

      HTML forms are essential elements for collecting user input on a webpage. They allow users to enter data and submit it to a server for processing. The <form> element wraps around input fields and buttons to group them together.

      • 1. Basic Structure of a Form

        A form typically consists of input fields (such as text boxes, checkboxes, radio buttons, etc.) and a submit button.

        Syntax:

        <form action="submit-form.php" method="POST">
    <!-- Input fields go here -->
</form>
        • action: Specifies the URL where the form data should be submitted.
        • method: Specifies the HTTP method to use when submitting the form. The common values are:
          • GET: Sends the form data as part of the URL (useful for searches).
          • POST: Sends the form data as part of the HTTP request body (more secure, commonly used).

        Example:

        <form action="/submit" method="POST">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name">

    <input type="submit" value="Submit">
</form>
      • 2. Form Attributes

        Here are some common attributes used with the <form> element:

        2.1 action

        Defines where the form data will be submitted (URL or file).

        2.2 method

        Defines how the form data will be sent (GET or POST).

        2.3 target

        Specifies where to display the response after form submission. The common values are:

        • _self: Loads the response in the same window (default).
        • _blank: Opens the response in a new tab or window.
        • _parent: Loads the response in the parent frame.
        • _top: Loads the response in the full body of the window.

        Example:

        <form action="/submit" method="POST" target="_blank">
    <!-- form elements -->
</form>

        2.4 autocomplete

        Specifies whether the browser should automatically complete form fields based on previous input. Values: on or off.

        Example:

        <form action="/submit" method="POST" autocomplete="on">
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" autocomplete="off">
</form>

        2.5 novalidate

        Disables form validation by the browser.

        Example:

        <form action="/submit" method="POST" novalidate>
    <!-- form elements -->
</form>
      • 3. Form Elements

        Forms consist of various types of input elements. Here are some of the key ones:

        3.1 Input Fields (<input>)

        As described earlier, input fields allow users to enter data. Examples include text boxes, passwords, email fields, etc.

        <label for="username">Username:</label>
<input type="text" id="username" name="username">

        3.2 Labels (<label>)

        The <label> element provides a caption for input fields and improves accessibility by associating a label with an input.

        <label for="password">Password:</label>
<input type="password" id="password" name="password">

        3.3 Textarea (<textarea>)

        The <textarea> element is used for multi-line text input.

        <label for="bio">Bio:</label>
<textarea id="bio" name="bio" rows="4" cols="50">Enter your bio...</textarea>

        3.4 Select Box (<select>)

        The <select> element creates a drop-down menu with options.

        <label for="country">Country:</label>
<select id="country" name="country">
    <option value="usa">USA</option>
    <option value="canada">Canada</option>
    <option value="uk">UK</option>
</select>

        3.5 Radio Buttons (<input type="radio">)

        Radio buttons allow users to select one option from a set.

        <label>Gender:</label>
<input type="radio" id="male" name="gender" value="male">
<label for="male">Male</label>
<input type="radio" id="female" name="gender" value="female">
<label for="female">Female</label>

        3.6 Checkboxes (<input type="checkbox">)

        Checkboxes allow users to select multiple options.

        <label for="subscribe">Subscribe to Newsletter:</label>
<input type="checkbox" id="subscribe" name="subscribe">

        3.7 Buttons

        The submit, reset, and button elements, as mentioned in the button section, are used to handle user actions.

        <input type="submit" value="Submit">
<input type="reset" value="Reset">
      • 4. Form Validation

        HTML forms include built-in validation to ensure that the submitted data meets the specified requirements.

        4.1 Required Fields

        Use the required attribute to make a field mandatory.

        <input type="text" name="username" required>

        4.2 Pattern Matching

        The pattern attribute allows you to specify a regular expression that the input must match.

        <input type="text" name="zipcode" pattern="\\d{5}" title="Enter a valid 5-digit ZIP code">

        4.3 Input Length

        The minlength and maxlength attributes define the minimum and maximum length of input.

        <input type="text" name="username" minlength="3" maxlength="15">

        4.4 Number Input Limits

        The min, max, and step attributes restrict the range of numeric inputs.

        <input type="number" name="age" min="18" max="100" step="1">
      • 5. Form Submission Methods

        Forms can submit data in two ways:

        5.1 GET Method

        • Sends form data appended to the URL as query parameters.
        • Useful for non-sensitive data (like search queries).

        Example:

        <form action="/search" method="GET">
    <input type="text" name="query">
    <input type="submit" value="Search">
</form>

        URL Example: /search?query=example

        5.2 POST Method

        • Sends form data in the request body.
        • Ideal for sensitive data (like login forms) since it is not visible in the URL.

        Example:

        <form action="/login" method="POST">
    <input type="text" name="username">
    <input type="password" name="password">
    <input type="submit" value="Login">
</form>
      • 6. File Upload Forms

        The enctype="multipart/form-data" attribute is required to handle file uploads.

        Example:

        <form action="/upload" method="POST" enctype="multipart/form-data">
    <label for="file">Choose a file:</label>
    <input type="file" id="file" name="file">
    <input type="submit" value="Upload">
</form>

  • CSS
    • 1. Introduction to CSS
      • Definition: CSS (Cascading Style Sheets) is used to control the presentation and layout of web pages.
      • Purpose: CSS separates content (HTML) from design, making it easier to manage the appearance of web pages across different devices and screen sizes.
      • Cascading: CSS applies styles according to a priority order of rules. If multiple rules apply to the same element, the most specific rule takes precedence.
    • 2. CSS Syntax and Structure

      CSS (Cascading Style Sheets) is written in rulesets that define how HTML elements should be displayed. The basic structure of a CSS rule includes selectors, properties, and values.

      • Basic CSS Rule Syntax:
        selector {
    property: value;
    property: value;
}
        • Selector: Specifies the HTML element(s) you want to style.
        • Property: Defines the style aspect you want to change (e.g., color, margin).
        • Value: The setting for the property (e.g., red, 20px).
      • 1. Selectors

        CSS selectors are used to target the HTML elements you want to apply styles to. There are different types of selectors, each designed to target elements in a specific way.

        • 1.1 Element Selector

          The element selector applies styles to all instances of the given HTML element.

          • Example:
            p {
    color: blue;
    font-size: 16px;
}

            This CSS rule targets all <p> (paragraph) elements, turning the text color blue and setting the font size to 16px.

        • 1.2 Class Selector

          The class selector applies styles to all elements with a specific class attribute. Class selectors start with a period (.).

          • Example:
            .card {
    border: 1px solid black;
    padding: 20px;
}

            This rule applies to any HTML element with the class="card", adding a black border and padding.

            HTML:

            <div class="card">This is a card.</div>
        • 1.3 ID Selector

          The ID selector targets a specific element with a unique id attribute. ID selectors are prefixed with a hash (#). IDs must be unique within a page.

          • Example:
            #header {
    background-color: grey;
    padding: 10px;
}

            This rule styles the element with id="header" by adding a grey background and padding.

            HTML:

    • 6. CSS Units

      CSS uses various units to define the size of elements, spacing, fonts, and more. These units can be classified into two categories: absolute and relative units.

      • 1. Absolute Units

        Absolute units represent fixed sizes, meaning they do not change based on the screen size, parent element, or any other external factors. These units are ideal when you want elements to have a precise size regardless of the display or device.

        Common Absolute Units:

        • px (Pixels): The most commonly used unit, representing one dot on the screen. It’s fixed and remains the same across devices.
        • cm (Centimeters): Unit measured in centimeters, useful for printed documents but not commonly used for screen display.
        • mm (Millimeters): Unit measured in millimeters.
        • in (Inches): Unit measured in inches, generally used for print media.
        • pt (Points): Used in typography, 1 point is 1/72 of an inch.
        • pc (Picas): Another unit used in typography; 1 pica equals 12 points.

        Example of Absolute Units:

        div {
    width: 300px;
    height: 200px;
    padding: 10cm;
    margin: 1in;
}
        • The width and height are set in pixels (px).
        • Padding is set in centimeters (cm).
        • Margin is set in inches (in).

        Absolute Unit Use Cases:

        • Useful for scenarios where precise sizes are required (e.g., fixed layout designs, print media).
        • Drawback: Not responsive, meaning elements won’t adjust well to different screen sizes.
      • 2. Relative Units

        Relative units are flexible and scale based on the size of other elements, like the parent element or the viewport (browser window). These units are critical for building responsive designs that adapt to different screen sizes and devices.

        Common Relative Units:

        1. em

        • Relative to the font-size of the element itself or the parent element.
        • 1em equals the current font size of the element. If the font size of a parent element is set to 16px, then 1em equals 16px.
        • Useful for creating elements that scale based on the parent font size.

        Example:

        p {
    font-size: 1.5em;  /* 1.5 times the parent font size */
}

        2. rem

        • Relative to the root element’s font-size (<html> element). By default, most browsers set the root font-size to 16px.
        • 1rem equals the font size of the root element, usually 16px.

        Example:

        h1 {
    font-size: 2rem;  /* 2 times the root font size, typically 32px */
}
        • Difference between em and rem: em scales based on the parent element, while rem is consistent and scales based on the root font size.

        3. % (Percentage)

        • Percentages are relative to the parent element’s size.
        • If you set width: 50%, the element will take up 50% of its parent element's width.

        Example:

        div {
    width: 50%;  /* Takes up 50% of the parent container's width */
}

        4. vw (Viewport Width)

        • 1vw equals 1% of the viewport’s width. The viewport is the visible area of the browser window.
        • As the window resizes, the vw value adjusts accordingly.

        Example:

        div {
    width: 50vw;  /* Takes up 50% of the viewport's width */
}

        5. vh (Viewport Height)

        • 1vh equals 1% of the viewport’s height.
        • Similar to vw, the vh unit adapts to the viewport’s height.

        Example:

        div {
    height: 100vh;  /* Full height of the viewport */
}

        6. vmin and vmax

        • vmin: Uses the smaller of either the viewport’s width or height. For example, if the viewport width is 800px and the height is 600px, 1vmin would be 6px (1% of 600px).
        • vmax: Uses the larger of either the viewport’s width or height. In the same case, 1vmax would be 8px (1% of 800px).

        Example:

        div {
    width: 50vmin;  /* 50% of the smaller dimension of the viewport */
}

        7. ex and ch

        • ex: Relative to the height of the lowercase letter "x" in the font.
        • ch: Relative to the width of the "0" (zero) character in the font. Useful for creating text inputs or monospaced designs.

        Example:

        input {
    width: 10ch;  /* Width equals the size of 10 "0" characters */
}

      In CSS, colors can be applied to elements using different notations. You can use predefined color names or specify colors using hexadecimal, RGB/RGBA, or HSL/HSLA values. Each method provides a different way to define colors, depending on your needs for accuracy, transparency, or flexibility.

        Hexadecimal (hex) notation represents colors using a 6-digit or 3-digit hexadecimal number. Each pair of digits represents the intensity of red, green, and blue, ranging from 00 (lowest) to FF (highest).

        Key Points:

        • More precise than named colors.
        • Often used for web design because of its wide support and flexibility.
        • Follows the format: #RRGGBB.

        h1 {

}

        In this example:

        • #ff0000: The first two digits (ff) represent red.
        • The second pair (00) represents green.
        • The third pair (00) represents blue.

}

      • 3. RGB and RGBA

        The RGB color model defines colors using three values: red, green, and blue. Each value ranges from 0 (none of that color) to 255 (maximum of that color). RGBA adds an additional alpha channel to control the color’s transparency.

        Key Points:

        • Allows more precise control of colors by setting individual values for red, green, and blue.
        • RGBA is useful for adding transparency, making it perfect for overlays and backgrounds.

        RGB Syntax:

        h2 {
    color: rgb(255, 0, 0);  /* Red */
}
        • rgb(255, 0, 0) produces a solid red color by setting the red value to 255, while green and blue are both set to 0.

        RGBA Syntax:

        div {
    background-color: rgba(255, 0, 0, 0.5);  /* Semi-transparent red */
}
        • rgba(255, 0, 0, 0.5) produces a red color with 50% opacity (alpha channel is 0.5, meaning 50% transparent).

        Alpha Channel Values:

        • The alpha value ranges from 0 (completely transparent) to 1 (completely opaque).
      • 4. HSL and HSLA

        The HSL color model represents colors using three values:

        • Hue: The type of color, ranging from 0 to 360 degrees on the color wheel (0 is red, 120 is green, 240 is blue).
        • Saturation: How pure the color is, from 0% (gray) to 100% (full color).
        • Lightness: How light or dark the color is, from 0% (black) to 100% (white).

        HSLA adds an alpha channel for transparency, just like RGBA.

        Key Points:

        • HSL is useful when you want to work with color theory, as you can easily adjust hue, saturation, and lightness.
        • HSLA adds transparency control.

        HSL Syntax:

        p {
    color: hsl(0, 100%, 50%);  /* Red */
}
        • hsl(0, 100%, 50%): This produces a pure red color.
          • Hue: 0 (Red).
          • Saturation: 100% (full intensity).
          • Lightness: 50% (normal brightness).

        HSLA Syntax:

        div {
    background-color: hsla(0, 100%, 50%, 0.5);  /* Semi-transparent red */
}
        • hsla(0, 100%, 50%, 0.5) produces a semi-transparent red, with 50% opacity.

    • 8. CSS Fonts and Text Styling

      CSS offers a variety of properties to control how text appears on a webpage. These properties allow you to define fonts, control the size, weight, style, alignment, and more. Proper text styling is crucial for both readability and design.

        p {

}

        The font-size property controls the size of the text. You can define font sizes using various units, including px (pixels), em (relative to the parent element), rem (relative to the root element), or %.

        Syntax:

        font-size: 16px;
        • Absolute Units: px for fixed sizing.
        • Relative Units: em or rem for scalable sizing, making the design more responsive.

        Example:

        h1 {
    font-size: 2em;  /* 2 times the size of the parent font size */
}
      • 3. Font Weight

        The font-weight property specifies the thickness of the text. You can use predefined values such as normal or bold, or numeric values ranging from 100 to 900, where 400 is normal and 700 is bold.

        Syntax:

        font-weight: bold;
        • Lighter Text: Use values less than 400.
        • Heavier Text: Use values greater than 400.

        Example:

        h2 {
    font-weight: 700;  /* Bold text */
}
      • 4. Font Style

        The font-style property controls the style of the font. Common values are normal, italic, and oblique.

        • Normal: The default, upright text.
        • Italic: A slanted version of the text.
        • Oblique: Similar to italic but can be artificially slanted.

        Syntax:

        font-style: italic;

        Example:

        blockquote {
    font-style: italic;
}

        The text-align property specifies the horizontal alignment of the text within its container. The common values are left, right, center, and justify.

        • Left: Aligns text to the left (default for left-to-right languages).
        • Right: Aligns text to the right.
        • Center: Centers the text within the element.
        • Justify: Spreads the text so that each line has equal width, with straight edges on both sides.

        Syntax:

        text-align: center;

        Example:

        p {
    text-align: justify;  /* Full-width justified text */
}

        The text-decoration property adds decorative elements to the text, such as underlining, overlining, or striking through the text.

        • underline: Adds a line below the text.
        • line-through: Draws a line through the middle of the text.

        text-decoration: underline;

        a {
    text-decoration: none;  /* Removes the underline from links */

        Syntax:

        line-height: 1.5;  /* 1.5 times the font size */
        • Unitless Value: Scales proportionally with the font size.
        • Percentage or Unit Value: Provides more precise control.

        p {
    line-height: 1.6;  /* Adds more spacing between lines */
}

        The text-transform property changes the case of text. The common values are uppercase, lowercase, and capitalize.

        • Uppercase: Converts all text to uppercase.
        • Lowercase: Converts all text to lowercase.

        text-transform: uppercase;

        div {
    background-color: lightblue;

        • repeat: The image repeats both horizontally and vertically (default behavior).

        background-repeat: no-repeat;

        Example:

        div {
    background-image: url('background.jpg');
    background-repeat: no-repeat;
}

        This prevents the background image from repeating.

      • 4. Background Size

        The background-size property defines the size of the background image. It controls whether the image stretches, resizes, or maintains its original dimensions.

        Values:

        • auto: Keeps the original size of the image (default).
        • cover: Scales the image to cover the entire element, maintaining aspect ratio.
        • contain: Scales the image to be fully contained within the element, maintaining aspect ratio.
        • Specific values: You can also specify sizes using units like px, %, or em.

        Syntax:

        background-size: cover;

        Example:

        div {
    background-image: url('background.jpg');
    background-size: cover;
}

        This scales the background image to cover the entire div without distorting it.

      • 5. Background Position

        The background-position property sets the starting position of the background image within the element. You can position the background using keywords or specific values.

        Values:

        • Keywords: top, bottom, left, right, center.
        • Coordinates: You can also use values like 10px 20px (first value for horizontal, second for vertical positioning).
        • Percentage: Positioning with percentages (e.g., 50% 50%) places the image at a relative position within the element.

        Syntax:

        background-position: center;

        Example:

        div {
    background-image: url('background.jpg');
    background-position: top right;
}

        This places the background image at the top-right corner of the div.

      • 6. Background Attachment

        The background-attachment property controls whether the background image scrolls with the rest of the content or stays fixed in place.

        Values:

        • scroll: The background image moves as the user scrolls the page (default behavior).
        • fixed: The background image stays fixed in place, even when the user scrolls.
        • local: The background image scrolls with the element's content (useful for scrollable elements).

        Syntax:

        background-attachment: fixed;

        Example:

        div {
    background-image: url('background.jpg');
    background-attachment: fixed;
}

        This makes the background image remain fixed when the user scrolls.

          • Examples of inline elements: <span>, <a>, <strong>.

          • Takes only the necessary space for its content.

          Example:

          span {
    display: inline;
}
        • 1.3 inline-block

          • The parent becomes a grid container.

    • 13. CSS Positioning

      CSS positioning allows you to control how elements are placed in relation to the document, their parent elements, or the browser window. The positioning model controls how elements interact with other elements and how they are rendered on the page.

      • 1. Static Positioning
        • Static is the default position for all elements.
        • The element is positioned according to the normal flow of the document.
        • No special positioning is applied, and the element appears where it naturally would in the DOM.

        Key Points:

        • The element does not react to any top, left, right, or bottom properties.
        • This is the default behavior for all HTML elements.
        .static-element {
    position: static;  /* Default value */
    background-color: lightgray;
}
      • 2. Relative Positioning
        • Relative positioning allows you to position an element relative to its normal position.
        • It shifts the element from where it would normally be in the document flow.

        Key Points:

        • The element still occupies its original space in the document, but it’s visually offset by top, left, right, or bottom properties.
        • Useful for small adjustments.
        .relative-element {
    position: relative;
    top: 20px;   /* Moves the element 20px down from its normal position */
    left: 10px;  /* Moves the element 10px to the right from its normal position */
    background-color: lightgreen;
}
      • 3. Absolute Positioning
        • Absolute positioning removes an element from the normal document flow and positions it relative to its nearest positioned ancestor (an ancestor with position set to relative, absolute, fixed, or sticky).

        Key Points:

        • If there is no positioned ancestor, the element will be positioned relative to the initial containing block (usually the <html> element or the browser window).
        • The element does not affect the position of other elements, as it is taken out of the normal document flow.
        .absolute-element {
    position: absolute;
    top: 50px;  /* Moves the element 50px down from the nearest positioned ancestor */
    right: 20px;  /* Moves the element 20px from the right of its containing block */
    background-color: lightblue;
}
      • 4. Fixed Positioning
        • Fixed positioning removes the element from the normal flow and positions it relative to the browser window.

        Key Points:

        • The element will stay fixed in its position even when the page is scrolled.
        • Often used for navigation bars, headers, or floating buttons.
        .fixed-element {
    position: fixed;
    bottom: 10px;  /* Stays 10px from the bottom of the window, even on scroll */
    right: 10px;   /* Stays 10px from the right of the window */
    background-color: coral;
    padding: 10px;
}
      • 5. Sticky Positioning
        • Sticky positioning is a hybrid of relative and fixed positioning.
        • The element is positioned relative until it reaches a specified point in the viewport (based on scroll position), and then it becomes fixed.

        Key Points:

        • The element behaves like relative until a certain scroll point is reached, and then it sticks to the top, bottom, or side of the viewport.
        .sticky-element {
    position: sticky;
    top: 0;  /* Sticks to the top when the page is scrolled */
    background-color: lightcoral;
    padding: 10px;
}

    • 15. CSS Flexbox

      The Flexbox Layout Model provides an efficient way to align, distribute, and organize space among items in a container, especially when the size of the items is unknown or dynamic. Flexbox is particularly useful for building responsive layouts, navigation bars, and handling alignment of elements.

      • 1. Creating a Flex Container

        The flexbox layout starts by creating a flex container using the display: flex property. All direct children of the flex container automatically become flex items, and their layout is governed by the properties of the flexbox model.

        Syntax:

        .container {
    display: flex;
}

        This makes all direct children of .container flex items that are arranged according to the flexbox rules.

      • 2. Main Flexbox Properties
        • 2.1 flex-direction

          The flex-direction property defines the direction in which the flex items are placed in the flex container.

          Values:

          • row: Items are placed in a horizontal row (default).
          • row-reverse: Items are placed in a horizontal row but in reverse order.
          • column: Items are placed in a vertical column.
          • column-reverse: Items are placed in a vertical column but in reverse order.

          Example:

          .container {
    display: flex;
    flex-direction: row;
}
        • 2.2 flex-wrap

          The flex-wrap property controls whether the flex items should wrap onto multiple lines if there isn't enough space in the container.

          Values:

          • nowrap: All flex items are kept on a single line (default).
          • wrap: Flex items wrap onto multiple lines when necessary.
          • wrap-reverse: Items wrap onto multiple lines in reverse order.

          Example:

          .container {
    display: flex;
    flex-wrap: wrap;
}
        • 2.3 justify-content

          The justify-content property aligns flex items along the main axis (horizontal by default).

          Values:

          • flex-start: Items are aligned to the start of the flex container (default).
          • flex-end: Items are aligned to the end of the flex container.
          • center: Items are centered along the main axis.
          • space-between: Items are evenly distributed, with space between them.
          • space-around: Items are evenly distributed, with space around them.
          • space-evenly: Items are evenly distributed, with equal space between them and the container.

          Example:

          .container {
    display: flex;
    justify-content: center;
}
        • 2.4 align-items

          The align-items property aligns flex items along the cross axis (vertical by default).

          Values:

          • stretch: Items stretch to fill the container (default).
          • flex-start: Items are aligned to the start of the cross axis (top in a row).
          • flex-end: Items are aligned to the end of the cross axis (bottom in a row).
          • center: Items are centered along the cross axis.
          • baseline: Items are aligned along their baseline.

          Example:

          .container {
    display: flex;
    align-items: center;
}
        • 2.5 align-content

          The align-content property controls how multiple lines of flex items are spaced if there is extra space in the cross axis. This property only applies if flex-wrap is enabled and there are multiple lines of items.

          Values:

          • stretch: Lines stretch to fill the container (default).
          • flex-start: Lines are packed toward the start of the container.
          • flex-end: Lines are packed toward the end of the container.
          • center: Lines are centered in the container.
          • space-between: Lines are evenly distributed with no space at the edges.
          • space-around: Lines are evenly distributed with space around them.

          Example:

          .container {
    display: flex;
    flex-wrap: wrap;
    align-content: space-between;
}
      • 3. Flex Item Properties

        These properties apply to the individual flex items within the flex container.

        • 3.1 order

          The order property controls the order of the flex items. By default, all items have a value of 0. You can change the order by assigning positive or negative values.

          Example:

          .item {
    order: 2;
}
        • 3.2 align-self

          The align-self property allows an individual flex item to override the align-items property. This property aligns the flex item along the cross axis.

          Values:

          • auto: Uses the align-items value from the parent container (default).
          • flex-start: Aligns the item to the start of the cross axis.
          • flex-end: Aligns the item to the end of the cross axis.
          • center: Centers the item along the cross axis.
          • baseline: Aligns the item along its baseline.
          • stretch: Stretches the item to fill the container.

          Example:

          .item {
    align-self: flex-end;
}
        • 3.3 flex-grow

          The flex-grow property defines how much an item will grow relative to other items inside the flex container. The value must be a positive number (default is 0).

          Example:

          .item {
    flex-grow: 2;  /* This item will grow twice as much as items with flex-grow: 1 */
}
        • 3.4 flex-shrink

          The flex-shrink property defines how much an item will shrink relative to other items when there is not enough space in the flex container. The value must be a positive number (default is 1).

          Example:

          .item {
    flex-shrink: 1;  /* Item will shrink if needed */
}
        • 3.5 flex-basis

          The flex-basis property specifies the initial size of a flex item before any growing or shrinking occurs. It can be set in units like px, %, or auto.

          Example:

          .item {
    flex-basis: 200px;
}

      The CSS Grid Layout Model provides a two-dimensional system for building web layouts. With grid, you can align and distribute elements in both rows and columns, making it a powerful tool for creating complex layouts.

      • 1. Creating a Grid Container

        To use the grid layout, you must first define a grid container by applying display: grid to a parent element. All direct children of this container become grid items.

        Syntax:

        .container {
    display: grid;
}

        This makes all direct children of .container grid items that are arranged according to the grid rules.

      • 2. Main Grid Properties
        • 2.1 grid-template-columns

          The grid-template-columns property defines the number of columns in the grid and their respective widths. You can specify the width using units like px, fr (fractional unit), %, or auto.

          Syntax:

          grid-template-columns: 200px 1fr 1fr;

          In this example:

          • The first column is 200px wide.
          • The second and third columns take up equal space, splitting the remaining space evenly using 1fr.
        • 2.2 grid-template-rows

          The grid-template-rows property defines the number of rows in the grid and their respective heights.

          Syntax:

          grid-template-rows: 100px auto 50px;

          In this example:

          • The first row is 100px high.
          • The second row will automatically adjust its height based on content.
          • The third row is 50px high.
        • 2.3 grid-column and grid-row

          These properties allow individual grid items to span across multiple columns or rows.

          Syntax:

          grid-column: 1 / 3;
grid-row: 1 / 2;

          In this example:

          • grid-column: 1 / 3: The item will start in the first column and span to the third column, covering both columns 1 and 2.
          • grid-row: 1 / 2: The item will span across the first row only.
        • 2.4 grid-gap (or gap)

          The grid-gap (or simply gap) property defines the spacing between rows and columns in the grid.

          Syntax:

          grid-gap: 20px;

          This creates a 20px gap between each grid row and column.

          You can also specify different values for row and column gaps:

          grid-gap: 10px 20px;

          In this example:

          • 10px gap between rows.
          • 20px gap between columns.
        • 2.5 align-items

          The align-items property controls how grid items are aligned vertically along the block axis (the column).

          Values:

          • stretch: Items stretch to fill the grid area (default).
          • start: Items are aligned to the top of the grid area.
          • end: Items are aligned to the bottom of the grid area.
          • center: Items are centered vertically in the grid area.

          Syntax:

          align-items: center;
        • 2.6 justify-items

          The justify-items property controls how grid items are aligned horizontally along the inline axis (the row).

          Values:

          • start: Items are aligned to the left of the grid area.
          • end: Items are aligned to the right of the grid area.
          • center: Items are centered horizontally in the grid area.
          • stretch: Items stretch to fill the grid area (default).

          Syntax:

          justify-items: center;
        • Combined Example

          Here’s an example that demonstrates how to create a basic grid layout:

          HTML:

          <!DOCTYPE html>

<head>

            display: grid;
            grid-template-columns: 200px 1fr 1fr;

            justify-items: center;

          • Item 2 spans across the last two rows (grid-row: 2 / 4).
          • Other items are placed in their respective grid cells.
          • All items are centered both vertically and horizontally (align-items: center, justify-items: center).
        • Summary of CSS Grid
          • Grid Layout Model: A two-dimensional layout system that allows you to position elements in rows and columns.
            • display: grid creates a grid container.
            • Main Properties:
              • grid-template-columns: Defines the number and size of columns.

    • 17. CSS Transitions and Animations

      CSS provides two methods for creating visual effects:

      • Transitions: Allow for smooth changes between two states.
      • Animations: Provide more complex and multi-step control over how CSS properties change over time.
      • 1. CSS Transitions

        Transitions allow you to change property values over a specified duration, making the change smooth rather than instantaneous. They are ideal for hover effects, focusing elements, or any change in state that can be animated over time.

        Key Transition Properties:

        1.1 transition-property

        Specifies which CSS property or properties will transition (e.g., width, height, color, background-color).

        1.2 transition-duration

        Specifies the duration over which the transition should occur (e.g., 0.5s for half a second).

        1.3 transition-timing-function

        Specifies the speed curve of the transition. Common values include:

        • ease: Starts slow, speeds up, then slows down (default).
        • linear: Changes at a constant speed.
        • ease-in: Starts slow, then speeds up.
        • ease-out: Starts fast, then slows down.
        • ease-in-out: Starts and ends slowly, with a fast middle.

        1.4 transition-delay

        Specifies a delay before the transition starts.

        Syntax:

        transition: property duration timing-function delay;

        Example:

        button {
    background-color: lightblue;
    transition: background-color 0.3s ease-in-out, transform 0.3s ease-in-out;
}

button:hover {
    background-color: coral;
    transform: scale(1.1);  /* Slightly enlarge on hover */
}

        In this example:

        • The background-color and transform properties will transition over 0.3 seconds with an ease-in-out timing function.
        • On hover, the button changes its color and slightly enlarges smoothly.
      • 2. CSS Animations

        Animations provide more control than transitions, allowing you to define multiple keyframes for more complex movements or transformations. Animations can loop, reverse, and include intermediate states.

        Key Animation Properties:

        2.1 Keyframes

        @keyframes define the sequence of steps in the animation. Each keyframe represents a percentage of the animation's duration, with 0% being the start and 100% the end.

        Syntax:

        @keyframes animation-name {
    0% {
        /* Starting styles */
    }
    50% {
        /* Midpoint styles */
    }
    100% {
        /* End styles */
    }
}

        2.2 animation-name

        Specifies the name of the keyframes animation to apply to an element.

        2.3 animation-duration

        Defines how long the animation will run (e.g., 2s for 2 seconds).

        2.4 animation-timing-function

        Controls how the animation progresses (similar to transition-timing-function).

        2.5 animation-delay

        Specifies a delay before the animation starts.

        2.6 animation-iteration-count

        Defines how many times the animation should repeat. You can use infinite to make it repeat forever.

        Syntax:

        animation: animation-name duration timing-function delay iteration-count;

        Example:

        @keyframes slide {
    0% {
        transform: translateX(0);
    }
    50% {
        transform: translateX(50px);
    }
    100% {
        transform: translateX(0);
    }
}

.box {
    width: 100px;
    height: 100px;
    background-color: lightgreen;
    animation: slide 2s ease-in-out infinite;
}

        In this example:

        • The box moves horizontally back and forth using the slide animation.
        • The animation lasts for 2 seconds, uses an ease-in-out timing function, and loops infinitely.
      • Key Differences Between Transitions and Animations
        FeatureTransitionsAnimations
        ControlCan only animate between two statesCan animate through multiple keyframes
        TriggerRequires a trigger (e.g., hover, focus)Can start automatically
        ComplexitySimple, limited to one state changeComplex, with full control over keyframes
        LoopingNo loopingCan loop with animation-iteration-count
        DelaySupported with transition-delaySupported with animation-delay

      • Summary of CSS Transitions and Animations
        • Transitions: Smoothly change properties between two states, ideal for hover effects or simple changes.
          • transition-property: Specifies which property to animate.
          • transition-duration: Duration of the transition.
          • transition-timing-function: Defines the speed curve.

          • Keyframes: Define the states of an element at various points in time.
          • animation-name: Specifies the keyframes animation to apply.
          • animation-duration: Specifies how long the animation runs.
          • animation-iteration-count: Defines how many times the animation repeats.
    • 18. CSS Media Queries
      • 1. What Are Media Queries?

        Media queries allow you to apply CSS rules based on the characteristics of the user’s device or viewport. They are commonly used to adjust layouts, font sizes, and other styles to fit different screen sizes.

        Syntax:

        @media (condition) {
    /* CSS rules here */
}
      • 2. Common Media Query Breakpoints

        Media queries typically rely on breakpoints, which are specific screen widths where the layout changes to accommodate the device. The most common breakpoints are defined using min-width and max-width to target various device sizes.

        Example of Breakpoints:

        Device TypeCommon Breakpoint
        Mobilemax-width: 600px
        Tabletmin-width: 600px and max-width: 768px
        Desktopmin-width: 768px

        Example:

        /* Styles for mobile devices (max width of 600px) */
@media (max-width: 600px) {
    body {
        background-color: lightblue;
    }
}

/* Styles for tablets (width between 600px and 768px) */
@media (min-width: 600px) and (max-width: 768px) {
    body {
        background-color: lightgreen;
    }
}

/* Styles for desktops (min width of 768px) */
@media (min-width: 768px) {
    body {
        background-color: lightcoral;
    }
}

        In this example:

        • If the screen width is less than 600px (e.g., on a mobile phone), the background color will be light blue.
        • For tablet-sized screens (between 600px and 768px), the background color will be light green.
        • For screens larger than 768px (e.g., desktops), the background will be light coral.
      • 3. Media Features

        Media queries can target several device characteristics or media features, such as width, height, orientation, and more.

        Common Media Features:

        • width: The width of the viewport (browser window).
        • height: The height of the viewport.
        • orientation: Whether the device is in landscape or portrait mode.
        • aspect-ratio: The ratio of the width to the height of the viewport.

        Example: Targeting Different Widths

        @media (min-width: 1024px) {
    /* Styles for screens larger than 1024px (desktops) */
    body {
        font-size: 18px;
    }
}

        Example: Targeting Device Orientation

        @media (orientation: portrait) {
    /* Styles for portrait orientation (e.g., phone held vertically) */
    body {
        background-color: yellow;
    }
}
      • 4. Media Query Operators

        You can combine media queries using logical operators like and, or, and not.

        and Operator

        • Combines multiple conditions, meaning all must be true for the styles to apply.

        Example:

        @media (min-width: 600px) and (orientation: landscape) {
    /* Styles for devices wider than 600px and in landscape mode */
    body {
        background-color: lightgray;
    }
}

        not Operator

        • Negates the query, applying the styles when the condition is not met.

        Example:

        @media not (min-width: 768px) {
    /* Styles for screens smaller than 768px */
    body {
        font-size: 14px;
    }
}
      • 5. Responsive Design with Media Queries

        The primary purpose of media queries is to implement responsive design, which allows your website to adapt gracefully to different screen sizes and devices. This ensures that the user experience is consistent and accessible across mobile, tablet, and desktop devices.

        Responsive Example:

        /* Mobile Styles */
@media (max-width: 600px) {
    .container {
        flex-direction: column;
        padding: 10px;
    }
}

/* Tablet Styles */
@media (min-width: 601px) and (max-width: 768px) {
    .container {
        flex-direction: row;
        padding: 20px;
    }
}

/* Desktop Styles */
@media (min-width: 769px) {
    .container {
        flex-direction: row;
        padding: 40px;
    }
}

        In this example:

        • On mobile devices, the .container element will use a column layout and smaller padding.
        • On tablets, it switches to a row layout with slightly larger padding.
        • On desktops, it uses a row layout with the largest padding.
    • 19. CSS Variables (Custom Properties)

      CSS Variables, also known as Custom Properties, allow you to store values in one place and reuse them throughout your stylesheet. This makes your CSS more maintainable and easier to update, especially when dealing with common values like colors, fonts, or spacing.

      • 1. Defining CSS Variables

        CSS variables are defined using the -- prefix. They are generally defined at the :root level (which is similar to the <html> element) to make them globally accessible, but you can also define them inside any other selector for local use.

        Syntax:

        :root {
    --main-color: #3498db;
    --secondary-color: #2ecc71;
    --font-size: 16px;
    --padding: 10px;
}

        In this example:

        • -main-color: Defines a primary blue color.
        • -secondary-color: Defines a secondary green color.
        • -font-size: Stores a common font size.
        • -padding: Stores a padding value.

        The :root selector allows the variables to be accessible globally in your CSS.

      • 2. Using CSS Variables

        You can access a variable using the var() function. This is particularly useful for keeping your styles consistent across your design.

        Syntax:

        color: var(--main-color);
padding: var(--padding);

        Example:

        :root {
    --main-color: #3498db;
    --secondary-color: #2ecc71;
    --font-size: 16px;
    --padding: 10px;
}

body {
    font-size: var(--font-size);
    background-color: var(--main-color);
    color: white;
}

button {
    background-color: var(--secondary-color);
    padding: var(--padding);
    color: white;
    border: none;
    cursor: pointer;
}

        In this example:

        • The body element uses the custom font size and background color from the defined variables.
        • The button element uses the -secondary-color and -padding variables.
      • 3. Fallback Values

        If a variable is undefined or not supported by the browser, you can specify a fallback value directly in the var() function.

        Syntax:

        color: var(--undefined-variable, black);

        In this example:

        • If -undefined-variable is not defined, the color will default to black.
      • 4. Local Scope for Variables

        Variables can be scoped locally to specific elements. This means you can override global variables within specific elements, making it possible to create different styles for different parts of your page without conflicting with the global variables.

        Example:

        :root {
    --main-color: #3498db;
}

.header {
    --main-color: #e74c3c;  /* Local override */
}

.header {
    color: var(--main-color);  /* Red for header */
}

footer {
    color: var(--main-color);  /* Blue for footer */
}

        In this example:

        • The header element uses a locally defined -main-color of red, while the rest of the document uses the global blue color.

    • 20. CSS Z-Index

      The z-index property controls the stacking order of elements when they overlap. Higher z-index values are positioned above lower values, determining which element appears on top in the stacking context.

      Key Points:

      • Z-Index only applies to elements with a position value other than static (i.e., relative, absolute, fixed, or sticky).
      • The default value is auto, meaning the element follows the normal stacking order.
      • Elements with a higher z-index will appear in front of elements with a lower z-index.

      Syntax:

      .element {
    position: relative;
    z-index: 10;
}

      Example:

      .box1 {
    position: absolute;
    z-index: 1;
    background-color: lightblue;
    width: 200px;
    height: 200px;
}

.box2 {
    position: absolute;
    z-index: 2;  /* Higher z-index, so it will appear on top */
    background-color: coral;
    width: 150px;
    height: 150px;
    top: 50px;
    left: 50px;
}

      In this example:

      • .box2 has a higher z-index (2) than .box1 (1), so it will appear on top of .box1 when they overlap.
    • 21. CSS Overflow

      The overflow property controls what happens when the content of an element exceeds the size of its container (i.e., when it "overflows"). It specifies whether to clip the content, add scrollbars, or allow content to overflow beyond the element's boundaries.

      Key Values:

      • 1. overflow: visible (default)
        • Content that exceeds the container’s bounds is visible and spills outside of the element.

        Example:

        .container {
    width: 200px;
    height: 100px;
    overflow: visible;  /* Content spills out of the container */
}
      • 2. overflow: hidden
        • Content that overflows the container is hidden and clipped. No scrollbars are added.

        Example:

        .container {
    width: 200px;
    height: 100px;
    overflow: hidden;  /* Content is clipped, no scrollbars */
}
      • 3. overflow: scroll
        • Scrollbars are always visible, even if the content fits inside the container. Both horizontal and vertical scrollbars will appear.

        Example:

        .container {
    width: 200px;
    height: 100px;
    overflow: scroll;  /* Scrollbars are always visible */
}
      • 4. overflow: auto
        • Scrollbars are only visible when necessary, meaning when the content exceeds the size of the container.

        Example:

        .container {
    width: 200px;
    height: 100px;
    overflow: auto;  /* Scrollbars appear only if content overflows */
}

          • Scope: Block-scoped (visible only within the { } block where it’s defined).
          • Hoisting: let is hoisted but remains uninitialized, so it cannot be accessed until it’s defined.
          • Use Case: Use let when you plan to reassign the variable’s value within the code.
          let city = "New York";
city = "Los Angeles"; // Reassignment is allowed
        • 1.3 const
          • Scope: Block-scoped, like let.
          • Assignment: const declares a constant variable, so you cannot reassign its value once defined.
          • Use Case: Use const for variables that should remain constant throughout the program.
          const country = "USA";
// country = "Canada"; // Error: Assignment to constant variable.
      • 2. Variable Naming Rules and Conventions
        • Naming Rules:
          • Variable names can include letters, digits, $, and _.
          • Names must start with a letter, $, or _.
          • Reserved keywords (like let, const, if, etc.) cannot be used as variable names.
        • Naming Conventions:
          • JavaScript conventionally uses camelCase for variable names (e.g., firstName, totalCost).
          • Constants that are deeply fixed and unchanging are sometimes written in uppercase with underscores (e.g., PI, MAX_COUNT).
        let firstName = "Alice";
const PI = 3.14159;
      • 3. Variable Scope
        • Global Scope: Variables declared outside of any function or block are globally scoped, accessible anywhere in the code.
        • Function Scope: Variables declared with var inside a function are only accessible within that function.
        • Block Scope: Variables declared with let or const inside a block { } are only accessible within that block.

        Example:

        let globalVar = "I’m global";

function example() {
    var functionScoped = "I’m in a function";
    if (true) {
        let blockScoped = "I’m in a block";
        console.log(blockScoped); // Accessible here
    }
    // console.log(blockScoped); // Error: blockScoped is not accessible here
}
      • 4. Hoisting in JavaScript Variables
        • Hoisting: JavaScript moves variable declarations to the top of their scope.
        • Var Hoisting: var declarations are hoisted and initialized with undefined.
        • Let and Const Hoisting: let and const are also hoisted, but remain uninitialized, leading to a temporal dead zone until they are declared.

        Example:

        console.log(name); // undefined, due to hoisting with `var`
var name = "Alice";

// console.log(city); // Error: Cannot access 'city' before initialization
let city = "New York";
      • Short Summary
        • Use let for reassignable, block-scoped variables.
        • Use const for constants that should not be reassigned.
        • Avoid var in modern JavaScript to prevent scoping issues and unexpected behavior due to hoisting.
        • Follow naming conventions for clarity and readability.
    • Data types
      • 1. Primitive Data Types

        JavaScript has seven primitive data types:

        • 1.1 String
          • Represents a sequence of characters.
          • Can be enclosed in single ('), double ("), or backtick (`) quotes.
            let name = "Alice";
        • 1.2 Number
          • Represents both integer and floating-point numbers.
          • There is no separate type for integers and decimals.
            let age = 25;
let temperature = 98.6;
        • 1.3 BigInt
          • Represents integers beyond the safe range for Number.
          • Created by appending n to the end of an integer.
            let largeNumber = 123456789012345678901234567890n;
        • 1.4 Boolean
          • Represents true or false values, useful for conditions.
            let isStudent = true;
        • 1.5 Undefined
          • A variable declared but not assigned a value is undefined.
            let city;
console.log(city); // Output: undefined
        • 1.6 Null
          • Represents an intentional absence of value.
            let job = null;
        • 1.7 Symbol
          • Represents a unique and immutable value, often used as object property identifiers.
            const uniqueId = Symbol("id");
      • 2. Complex Data Types
        • 2.1 Object
          • Used to store collections of data or more complex entities.
          • Objects contain key-value pairs, where values can be any type, including other objects.
            const person = {
    name: "Alice",
    age: 25,
    isStudent: true
};
        • 2.2 Array (a type of Object)
          • A list-like object that stores multiple values, indexed by numbers.
            let fruits = ["Apple", "Banana", "Cherry"];
      • 3. Type Checking

        You can check the data type of a value using the typeof operator:

        console.log(typeof 420000000000000215545121564);    // "number"
console.log(typeof 420000000000000215545121564);    // "bigint"
console.log(typeof "Hello");      // "string"
console.log(typeof true);         // "boolean"
console.log(typeof undefined);    // "undefined"
console.log(typeof null);         // "object" (this is a known quirk in JavaScript)
console.log(typeof Symbol());     // "symbol"
console.log(typeof {name: "Alice"}); // "object"
console.log(typeof [1, 2, 3]);    // "object"
      • 4. Type Conversion

        JavaScript allows implicit and explicit type conversion between types.

        Implicit Conversion

        • JavaScript automatically converts types in certain contexts, like adding a number to a string.
          let result = "5" + 10; // "510" (number 10 is converted to a string)

        Explicit Conversion

        • You can manually convert types using methods like String(), Number(), and Boolean().
          let num = "123";
let convertedNum = Number(num); // 123 as a number
