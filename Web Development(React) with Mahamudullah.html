  • HTML
    • Basic HTML Structure

      Open Visual Studio code. Create a html file in the folder you open. Then, type “!” and press enter button on your keyboard. A basic structure will be generate in there. Here is the code you will see:

      <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <!-- we will write code here. -->
</body>
</html>

      Document Structure:

      • <!DOCTYPE html>:
        • Declares the document as an HTML5 document. This tells the browser to render the page using the HTML5 standards.
      • <html lang="en">:
        • This is the root element of the HTML document. The lang="en" attribute specifies the language of the document (in this case, English).
      • <head>:
        • Contains meta-information about the document, such as the title, character encoding, and responsive behavior.
      • Inside the <head>:
        • <meta charset="UTF-8">:
          • This tag specifies the character encoding for the document as UTF-8, which is the most widely used encoding. It ensures that the webpage can display a wide range of characters from different languages correctly.
        • <meta name="viewport" content="width=device-width, initial-scale=1.0">:
          • This tag ensures the webpage is responsive on mobile devices. It controls the page's dimensions and scaling, setting the width to the device's width and ensuring a 1:1 scale (no zoom by default). Without this, mobile browsers may display your webpage incorrectly or zoomed out.
        • <title>Document</title>:
          • Defines the title of the document that will appear in the browser tab or title bar. It's also used by search engines to display the page's title in search results.
      • <body>:
        • This contains all the visible content of the webpage, such as text, images, links, buttons, and other elements. Everything you want users to see or interact with will be placed inside this element.

    • Anchor <a> & image <img> tag

      1. Anchor (<a>) Tag:

      • The anchor tag defines a hyperlink, which is used to link from one page to another.
      • The href attribute specifies the URL of the page the link goes to.

      Example:

      <a href="<https://www.example.com>">Visit Example Website</a>

      This creates a clickable link that takes the user to https://www.example.com.

      2. Image (<img>) Tag:

      • The image tag embeds an image in the page.
      • The src attribute specifies the image source (URL or path to the image file), and the alt attribute provides alternative text if the image cannot be displayed.

      Example:

      <img src="<https://via.placeholder.com/150>" alt="Placeholder Image">

      This embeds an image from the provided URL, and if the image fails to load, the alt text "Placeholder Image" will be displayed.

      3. Combining Anchor and Image:

      You can wrap an image in an anchor tag to make the image a clickable link.

      Example:

      <a href="<https://www.example.com>">
    <img src="<https://via.placeholder.com/150>" alt="Clickable Placeholder Image">
</a>

      This creates an image that, when clicked, takes the user to https://www.example.com.

    • Lists <li>

      HTML Lists

      Lists are a great way to group related items together in a structured manner. In HTML, there are three types of lists: ordered lists, unordered lists, and definition lists.

      • 1. Unordered List (<ul>)

        An unordered list displays items as bullet points. It is used when the order of items is not important.

        Syntax:

        <ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</ul>

        Example:

        <ul>
    <li>Apples</li>
    <li>Bananas</li>
    <li>Cherries</li>
</ul>

        Output:

        • Apples
        • Bananas
        • Cherries
      • 2. Ordered List (<ol>)

        An ordered list displays items in a specific sequence, typically with numbers. It is used when the order of items matters.

        Syntax:

        <ol>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ol>

        Example:

        <ol>
    <li>Step 1: Open the browser.</li>
    <li>Step 2: Navigate to the website.</li>
    <li>Step 3: Start exploring.</li>
</ol>

        Output:

        1. Step 1: Open the browser.
        1. Step 2: Navigate to the website.
        1. Step 3: Start exploring.

        Types of Ordered Lists:

        You can change the number format using the type attribute:

        • Numbers (default): <ol type="1">
        • Uppercase Letters: <ol type="A">
        • Lowercase Letters: <ol type="a">
        • Uppercase Roman Numerals: <ol type="I">
        • Lowercase Roman Numerals: <ol type="i">

        Example:

        <ol type="A">
    <li>Introduction</li>
    <li>Content</li>
    <li>Conclusion</li>
</ol>

        Output:

        A. Introduction

        B. Content

        C. Conclusion

      • 3. Definition List (<dl>)

        A definition list is used for terms and descriptions. It contains <dt> (definition term) and <dd> (definition description) elements.

        Syntax:

        <dl>
    <dt>HTML</dt>
    <dd>The standard language for creating web pages.</dd>

    <dt>CSS</dt>
    <dd>A style sheet language used for describing the look of a webpage.</dd>
</dl>

        Example:

        <dl>
    <dt>API</dt>
    <dd>Application Programming Interface</dd>

    <dt>HTTP</dt>
    <dd>Hypertext Transfer Protocol</dd>
</dl>

        Output:

        • API: Application Programming Interface
        • HTTP: Hypertext Transfer Protocol
      • Customizing List Style

        You can modify the appearance of lists using CSS.

        Changing Bullet Style in Unordered List:

        ul {
    list-style-type: square; /* options: circle, disc (default), square, none */
}

        Example:

        <ul style="list-style-type: square;">
    <li>Item 1</li>
    <li>Item 2</li>
</ul>

        Output:

        ■ Item 1

        ■ Item 2

    • Input <input>

      The <input> element in HTML is used to create interactive controls in web forms, allowing users to input data. It is a versatile element that supports various input types, such as text, passwords, email, numbers, etc.

      1. Basic Syntax of Input

      <input type="text" name="inputName">
      • type: Specifies the type of input (e.g., text, email, password).
      • name: Assigns a name to the input field (used in form submission).

      2. Common Input Types

      Below are the most commonly used input types:

      • 2.1 Text Input (type="text")

        Creates a single-line text input field for short, single-line data like names or titles.

        <label for="username">Username:</label>
<input type="text" id="username" name="username">
      • 2.2 Password Input (type="password")

        Hides the characters entered in the field (commonly used for passwords).

        <label for="password">Password:</label>
<input type="password" id="password" name="password">
      • 2.3 Email Input (type="email")

        Validates the input to ensure it's a properly formatted email address.

        <label for="email">Email:</label>
<input type="email" id="email" name="email">
      • 2.4 Number Input (type="number")

        Allows users to input numbers. You can also specify a range using min, max, and step.

        <label for="age">Age:</label>
<input type="number" id="age" name="age" min="1" max="100" step="1">
      • 2.5 Checkbox Input (type="checkbox")

        Creates a checkbox that allows the user to select multiple options.

        <label for="subscribe">Subscribe to Newsletter:</label>
<input type="checkbox" id="subscribe" name="subscribe">
      • 2.6 Radio Input (type="radio")

        Creates radio buttons that allow the user to select only one option from a group.

        <label for="gender">Gender:</label>
<input type="radio" id="male" name="gender" value="male"> Male
<input type="radio" id="female" name="gender" value="female"> Female
      • 2.7 Date Input (type="date")

        Provides a date picker for selecting a date.

        <label for="dob">Date of Birth:</label>
<input type="date" id="dob" name="dob">
      • 2.8 Range Input (type="range")

        Creates a slider control for selecting a value within a range.

        <label for="volume">Volume:</label>
<input type="range" id="volume" name="volume" min="0" max="100">
      • 2.9 File Input (type="file")

        Allows users to upload files.

        <label for="resume">Upload Resume:</label>
<input type="file" id="resume" name="resume">
      • 2.10 Submit Button (type="submit")

        Creates a button that submits the form.

        <input type="submit" value="Submit">
      • 2.11 Reset Button (type="reset")

        Resets all form fields to their initial values.

        <input type="reset" value="Reset">

      3. Input Attributes

      Here are some common attributes you can use with <input> elements:

      • 3.1 placeholder

        Provides a hint to the user about what to enter in the input field.

        <input type="text" placeholder="Enter your name">
      • 3.2 required

        Makes an input field mandatory for form submission.

        <input type="email" required>
      • 3.3 disabled

        Disables an input field, making it non-editable and non-submittable.

        <input type="text" disabled>
      • 3.4 readonly

        Makes an input field read-only, so users can view but not change the content.

        <input type="text" value="This is read-only" readonly>
      • 3.5 value

        Sets the initial value of an input field.

        <input type="text" value="Pre-filled Text">
      • 3.6 maxlength

        Limits the number of characters the user can input.

        <input type="text" maxlength="10">
      • 3.7 min and max

        Specifies the minimum and maximum values for number, date, and range inputs.

        <input type="number" min="1" max="10">
<input type="date" min="2022-01-01" max="2024-12-31">
      • 3.8 step

        Defines the stepping interval for numeric or range input.

        <input type="number" step="5">
<input type="range" step="10">
    • Form <>

      HTML forms are essential elements for collecting user input on a webpage. They allow users to enter data and submit it to a server for processing. The <form> element wraps around input fields and buttons to group them together.

      • 1. Basic Structure of a Form

        A form typically consists of input fields (such as text boxes, checkboxes, radio buttons, etc.) and a submit button.

        Syntax:

        <form action="submit-form.php" method="POST">
    <!-- Input fields go here -->
</form>
        • action: Specifies the URL where the form data should be submitted.
        • method: Specifies the HTTP method to use when submitting the form. The common values are:
          • GET: Sends the form data as part of the URL (useful for searches).
          • POST: Sends the form data as part of the HTTP request body (more secure, commonly used).

        Example:

        <form action="/submit" method="POST">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name">

    <input type="submit" value="Submit">
</form>
      • 2. Form Attributes

        Here are some common attributes used with the <form> element:

        2.1 action

        Defines where the form data will be submitted (URL or file).

        2.2 method

        Defines how the form data will be sent (GET or POST).

        2.3 target

        Specifies where to display the response after form submission. The common values are:

        • _self: Loads the response in the same window (default).
        • _blank: Opens the response in a new tab or window.
        • _parent: Loads the response in the parent frame.
        • _top: Loads the response in the full body of the window.

        Example:

        <form action="/submit" method="POST" target="_blank">
    <!-- form elements -->
</form>

        2.4 autocomplete

        Specifies whether the browser should automatically complete form fields based on previous input. Values: on or off.

        Example:

        <form action="/submit" method="POST" autocomplete="on">
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" autocomplete="off">
</form>

        2.5 novalidate

        Disables form validation by the browser.

        Example:

        <form action="/submit" method="POST" novalidate>
    <!-- form elements -->
</form>
      • 3. Form Elements

        Forms consist of various types of input elements. Here are some of the key ones:

        3.1 Input Fields (<input>)

        As described earlier, input fields allow users to enter data. Examples include text boxes, passwords, email fields, etc.

        <label for="username">Username:</label>
<input type="text" id="username" name="username">

        3.2 Labels (<label>)

        The <label> element provides a caption for input fields and improves accessibility by associating a label with an input.

        <label for="password">Password:</label>
<input type="password" id="password" name="password">

        3.3 Textarea (<textarea>)

        The <textarea> element is used for multi-line text input.

        <label for="bio">Bio:</label>
<textarea id="bio" name="bio" rows="4" cols="50">Enter your bio...</textarea>

        3.4 Select Box (<select>)

        The <select> element creates a drop-down menu with options.

        <label for="country">Country:</label>
<select id="country" name="country">
    <option value="usa">USA</option>
    <option value="canada">Canada</option>
    <option value="uk">UK</option>
</select>

        3.5 Radio Buttons (<input type="radio">)

        Radio buttons allow users to select one option from a set.

        <label>Gender:</label>
<input type="radio" id="male" name="gender" value="male">
<label for="male">Male</label>
<input type="radio" id="female" name="gender" value="female">
<label for="female">Female</label>

        3.6 Checkboxes (<input type="checkbox">)

        Checkboxes allow users to select multiple options.

        <label for="subscribe">Subscribe to Newsletter:</label>
<input type="checkbox" id="subscribe" name="subscribe">

        3.7 Buttons

        The submit, reset, and button elements, as mentioned in the button section, are used to handle user actions.

        <input type="submit" value="Submit">
<input type="reset" value="Reset">
      • 4. Form Validation

        HTML forms include built-in validation to ensure that the submitted data meets the specified requirements.

        4.1 Required Fields

        Use the required attribute to make a field mandatory.

        <input type="text" name="username" required>

        4.2 Pattern Matching

        The pattern attribute allows you to specify a regular expression that the input must match.

        <input type="text" name="zipcode" pattern="\\d{5}" title="Enter a valid 5-digit ZIP code">

        4.3 Input Length

        The minlength and maxlength attributes define the minimum and maximum length of input.

        <input type="text" name="username" minlength="3" maxlength="15">

        4.4 Number Input Limits

        The min, max, and step attributes restrict the range of numeric inputs.

        <input type="number" name="age" min="18" max="100" step="1">
      • 5. Form Submission Methods

        Forms can submit data in two ways:

        5.1 GET Method

        • Sends form data appended to the URL as query parameters.
        • Useful for non-sensitive data (like search queries).

        Example:

        <form action="/search" method="GET">
    <input type="text" name="query">
    <input type="submit" value="Search">
</form>

        URL Example: /search?query=example

        5.2 POST Method

        • Sends form data in the request body.
        • Ideal for sensitive data (like login forms) since it is not visible in the URL.

        Example:

        <form action="/login" method="POST">
    <input type="text" name="username">
    <input type="password" name="password">
    <input type="submit" value="Login">
</form>
      • 6. File Upload Forms

        The enctype="multipart/form-data" attribute is required to handle file uploads.

        Example:

        <form action="/upload" method="POST" enctype="multipart/form-data">
    <label for="file">Choose a file:</label>
    <input type="file" id="file" name="file">
    <input type="submit" value="Upload">
</form>
